# 答辩后实验问题与改进方案

本文根据 2026-07-13 答辩后的组内讨论整理。原始对话同时混合了模型原理、实验缺陷和后续设想，下面按“问题是什么、当前项目处于什么状态、怎样解决、怎样证明解决有效”的顺序重新组织。

## 一、核心结论

目前实验最需要补强的不是继续无序增加 decoder，而是建立完整证据链：

1. 说明 PASTIS 输入、类别和样本分布。
2. 准确解释 Galileo 在哪里处理时间，以及时间维在哪里被压缩。
3. 用受控消融区分 encoder 的时序建模和 decoder 的时序融合。
4. 把物候先验转化为可验证的模型设计，而不是停留在口头描述。
5. 同时报告总体指标、分类别指标和可视化，分析模型具体错在哪里。
6. 用多 seed 或交叉验证说明结果是否稳定，并将模型提升与后处理提升分开报告。

优先级最高的是第二、三项。只有先解释清楚时间信息的流向，后续“物候是否有效”和“3D-Aware DPT 为什么有效”才有可信结论。

## 二、先统一 Galileo 时间维的表述

### 2.1 当前代码实际做了什么

项目输入是按月聚合的 12 时相 PASTIS 数据：

```text
[B, T=12, C=10, H=64, W=64]
```

Galileo 在 encoder 内部利用月份信息和时空 token 交互，因此各月份输出已经包含其他月份提供的上下文。**这不等于输出天然没有 T 维。** Galileo 的结构化 token 仍然可以恢复为逐月特征。

当前方案一至四为了对齐 Galileo 官方 segmentation linear probing 接口，采用：

```text
逐月上下文化特征
  [B, T=12, D=768, 16, 16]
        -> time mean
早期融合特征
  [B, D=768, 16, 16]
```

所以准确说法应为：

> Galileo encoder 已经进行时序上下文化；方案一至四在 encoder 输出到 decoder 的接口处，对上下文化后的逐月特征求均值，使 decoder 不再显式看到 T 维。

不能说“Galileo 没有处理时间”，也不应说“T 维是模型内部必然消失的”。时间均值是官方 probing 路线采用的下游读取策略，本项目早期融合实验沿用了该策略。

### 2.2 当前方案五与讨论中新实验的区别

当前 3D-Aware DPT（方案五）是：

```text
12个月一起进入 Galileo
  -> Galileo 内部进行跨时间交互
  -> 保留上下文化后的 T=12 特征
  -> 3D-Aware DPT 再进行显式时间融合
```

答辩讨论中学长提出的“一个时间一个时间送入 Galileo”则是另一个实验：

```text
每个月独立进入 Galileo
  -> encoder 内部没有跨月交互
  -> 堆叠12个月的独立特征
  -> 使用同一个时间 decoder 融合
```

两者不能混称为晚期融合。前者同时使用 Galileo 时序建模和 decoder 时序建模；后者刻意关闭 Galileo 的跨月能力，用来检验“时间关系应该主要由 encoder 学，还是主要由 decoder 学”。

### 2.3 建议的时间消融

固定数据划分、输入月份、Galileo 权重、loss、optimizer、有效 batch 和模型选择规则，仅改变时间处理位置：

| 实验 | Galileo 是否同时看到12个月 | decoder 是否看到 T | 目的 |
| --- | --- | --- | --- |
| T0：官方式 early mean | 是 | 否 | Galileo 官方 probing 对照 |
| T1：保留 T 的 3D-Aware DPT | 是 | 是 | 检验显式保留逐月特征是否有效 |
| T2：逐月独立 Galileo + 同一时间 decoder | 否 | 是 | 分离 encoder 跨月建模的贡献 |
| T3：T1 + 物候先验 | 是 | 是 | 检验领域先验是否提供额外收益 |

T1 已在当前项目中实现；T2 尚未实现。T2 应尽量复用 T1 的 decoder 和参数规模，否则无法把差异归因于 Galileo 的跨月交互。

## 三、问题与解决方法

### 3.1 输入、类别和样本量没有讲清楚

### 问题

答辩中只介绍了模型，没有清楚展示：

- 为什么输入是 12 个月、10 个波段和 `64x64` 子块。
- 19 个类别分别代表什么。
- 各类别在 train、val、test 中有多少样本。
- 类别不平衡到底有多严重。

“每类有多少图像”对语义分割并不足够，因为一张图可能包含多个类别。至少要同时统计 patch 数量和像素数量。

19 个有效 ID 的官方名称、中文含义以及哪些属于综合地类，已经整理在 [`docs/next.md`](next.md#pastis-的19个有效类别)。其中 ID 0 是非农业背景，ID 1、12、14、16、17 也不是单一植物品种；原始 ID 19 是忽略标签，不属于19个有效类别。

### 解决方法

增加一份只使用数据和标签的审计结果，按 fold 输出：

1. 样本和切分后 patch 总数。
2. 每类像素数、像素比例和包含该类的 patch 数。
3. void 像素比例。
4. 每月有效观测数量以及插值样本数量。
5. train、val、test 的类别分布差异。

报告中给出“类别编号、类别名、像素比例、patch 覆盖率、per-class IoU”总表。这样可以判断某个类别 IoU 低，是样本过少、跨 fold 分布漂移，还是模型本身分不清。

如果确认类别不平衡明显，可依次尝试：

- 对数反频率或 effective-number class weight，避免直接使用过大的倒数权重。
- Focal loss。
- 提高包含稀有类别 patch 的抽样概率。

三种方法必须单独做消融，不能一次全部加入。权重只能由 train fold 统计，不能使用 val/test 标签分布调参。

### 3.2 缺少物候描述和显式物候建模

### 问题

当前端到端模型可能学到季节规律，但答辩材料没有证明模型学到了什么，也没有显式利用不同作物在播种、生长和成熟期上的差异。因此“用了时序数据”还不能等价于“有效利用了物候”。

### 第一步：先做数据驱动的物候分析

不要直接硬编码“某作物只看某几个月”。先只使用 train fold：

1. 按类别统计每个月各波段的均值和标准差。
2. 计算 NDVI 等常用植被指数的月度曲线。
3. 计算每类与其他类别差异最大的月份或月份区间。
4. 对比 train 与 val 曲线，确认规律是否稳定。

统计和绘图必须使用真实月份编号；当前 crop-year 顺序为 9 月到次年 8 月，不能把数组下标 0 误写成自然年 1 月。

这比仅查询通用物候表更可靠，因为它对应 PASTIS 的年份、地区、传感器和类别定义。外部物候资料可以用于解释，但模型先验应优先从 train 数据估计。

### 第二步：从弱先验到强先验逐步加入模型

建议按以下顺序实验：

1. **月份位置编码**：让 decoder 明确知道每个特征对应哪个月份。当前方案五已经包含 month embedding。
2. **可学习时间注意力**：让模型对每个空间位置自适应选择月份，不固定窗口。
3. **类别条件时间注意力**：为不同类别学习不同的月份权重，直接对应“不同作物关注不同生育期”。
4. **物候软先验**：使用 train 数据得到的类别-月份曲线初始化或偏置注意力，而不是把其他月份硬删除。
5. **物候旁路**：将关键月份窗口单独编码，与完整时序主分支融合。

硬选择月份的风险是天气、播期和区域差异会造成物候漂移。优先使用 soft attention，并保留完整时序作为主分支。

### 第三步：可选的层级分类

可根据官方类别定义先划分粗类别，再预测细类别：

```text
共享时空特征
  -> 粗分类头：耕地/非耕地或作物大类
  -> 细分类头：19类
  -> 层级一致性约束
```

层级映射必须由可靠的作物分类依据确定，不能为了提高指标任意合并。该方案比时间消融和物候曲线分析优先级低。

### 3.3 train loss 下降，但 val loss 后期上升

### 问题

这不表示“模型还没有训练完”。当 train loss 持续下降，而 val loss 先下降后上升时，首先应判断为过拟合、概率校准变差或 train/val 分布差异。

val loss 与 mIoU 也可能走势不同：少数错误像素变得更加自信会明显抬高交叉熵，但 argmax 分割图变化不大，因此 mIoU 可能保持平台期。

### 已完成的措施

当前 Trainer 已经：

- 同时保存最低 `val_loss` 的 `best_val_loss.pt`。
- 保存最高 `val_mIoU` 的 `best_val_miou.pt`。
- 以 `val_mIoU` 执行早停。

最终 mIoU 报告应统一评估 `best_val_miou.pt`，不能使用最后一轮，也不能看 test 结果后再选择 checkpoint。

### 仍需补充的措施

1. 对每个主实验至少运行 3 个 seed，报告平均值和标准差。
2. 检查 val per-class IoU，确认下降主要来自哪些类别。
3. 单独尝试 dropout、weight decay、数据增强或更小 decoder，不要同时修改。
4. 对比最低 val loss checkpoint 与最高 val mIoU checkpoint 的置信度和分割边界。

如果进行五折交叉验证，应与论文固定 fold 协议分开报告：

- 与 Galileo 论文比较时，保持当前 train=`1/2/3`、val=`4`、test=`5`。
- 检查模型稳定性时，可以另做五折轮换并报告均值。

不能把五折均值直接当成论文 fold5 test 数值进行横向比较。

### 3.4 只有总体 mIoU，缺少分类别证据和可视化

### 问题

单一 mIoU 只能说明总体水平，不能回答：

- 哪些作物最容易混淆。
- 边界是否准确。
- 输出是否破碎。
- 稀有类别是否被多数类吞没。
- 多层 DPT 或 3D-Aware DPT 改善了什么。

### 解决方法

定量部分至少增加：

1. 19 类 per-class IoU。
2. confusion matrix。
3. 各类别 Precision、Recall 和 F1。
4. 边界 F1 或 boundary IoU。
5. 连通域数量、平均区域面积等破碎度指标。

定性可视化建议固定同一组 test 样本，横向展示：

```text
代表月份RGB | Ground Truth | 线性探测 | 早期融合DPT | 3D-Aware DPT | 错误图
```

样本选择规则应预先固定，包括：

- 固定随机 seed 抽样。
- 稀有类别样本。
- 按 mIoU 排序的最好、中位和最差样本。
- 边界复杂或预测破碎的样本。

不能只挑视觉效果最好的图片。每张图旁边简短说明主要混淆类别和可能原因，将可视化变成误差分析，而不只是展示。

### 3.5 是否需要更强或更新的预训练模型

### 问题

“Galileo 是不是目前最好”不能只根据发表时间回答，还需要明确比较范围：多光谱还是 RGB、单时相还是时序、是否支持 PASTIS 波段、参数量和预训练数据是否相同。

讨论中提到 MAESTRO 和 DINOv3-sat，只能先列为候选，不能直接写成更优结论。尤其要检查预训练数据是否包含 PASTIS 图像或高度重叠区域，否则可能形成数据泄漏或不公平优势。

### 解决方法

建立预训练模型审计表：

| 项目 | 需要记录的内容 |
| --- | --- |
| 输入模态 | RGB、多光谱、SAR、时序 |
| 时间处理 | 单时相或原生时序 |
| 预训练数据 | 数据集、地区、年份 |
| PASTIS 重叠风险 | 无、未知、确认重叠 |
| 参数量 | encoder 和 trainable head |
| 使用方式 | frozen linear probe 或相同 decoder |

先使用相同 fold 做 frozen linear probe，再考虑接入复杂 decoder。只使用 RGB 的模型改变了输入模态，适合作为扩展对照，不属于“输入和 encoder 固定，只比较 decoder”的主实验。

### 3.6 输出破碎与空间后处理

### 问题

农田通常具有较强空间连续性。如果预测内部出现大量孔洞或小碎片，纯像素 mIoU 之外还存在结构质量问题。

### 解决方法

先通过错误图和连通域指标确认是否真的存在系统性破碎，再按复杂度尝试：

1. 小连通域移除和邻域多数投票。
2. 形态学闭运算与孔洞填充。
3. 基于影像边界的 Dense CRF。
4. 若能获得可靠地块边界，进行 parcel 内概率投票。

所有阈值只能在 val 上选择，并同时报告原始输出和后处理输出。后处理不能掩盖模型缺陷，也不能把提升归到 decoder 本身。小地块可能被错误删除，因此必须观察 per-class 和不同地块尺度的影响。

## 四、建议实验矩阵

为了避免再次出现“方案很多但结论不清楚”，建议按以下顺序推进：

| 阶段 | 实验 | 主要回答的问题 |
| --- | --- | --- |
| A | 线性探测、方案一至四 | 更强空间 decoder 是否优于线性/卷积 baseline |
| B | 当前方案五 T1 | decoder 显式保留 T 是否有效 |
| C | 逐月独立 Galileo T2 | Galileo encoder 的跨月建模贡献多大 |
| D | T1 + 类别条件时间注意力 | 不同作物是否需要不同月份权重 |
| E | D + train-derived 物候软先验 | 显式领域知识是否进一步提升 |
| F | 最佳模型 + 后处理 | 空间连续性问题能否独立改善 |

每个实验统一记录：

- 配置文件和 Git commit。
- seed、参数量、显存和训练时间。
- best val mIoU 及其 epoch。
- fold5 test mIoU 和 per-class IoU。
- 至少一组固定样本可视化。
- 相比直接对照实验只改变了什么。

## 五、执行优先级

### P0：答辩后立即完成

1. 使用本文第二节统一 Galileo 时间处理的口径。
2. 完成类别、像素和 fold 分布统计。
3. 为已有模型输出 per-class IoU、confusion matrix 和固定样本可视化。
4. 当前正在训练的模型继续运行，不因整理工作修改其配置；统一使用 `best_val_miou.pt` 测试。

### P1：最关键的新实验

1. 完成当前保留 T 的 3D-Aware DPT 实验。
2. 新增“逐月独立 Galileo + 相同时间 decoder”消融。
3. 从 train fold 绘制作物月度光谱/植被指数曲线。
4. 在最佳时间模型上加入类别条件 soft temporal attention。

### P2：稳定性与扩展

1. 主实验运行多个 seed，报告均值和标准差。
2. 视时间和算力另做五折稳定性实验。
3. 审计 MAESTRO、DINOv3-sat 等候选 encoder 的输入兼容性和数据泄漏风险。
4. 对确认存在的预测破碎问题进行独立后处理实验。

## 六、答辩中可以使用的简洁表述

关于时间平均：

> Galileo 在 encoder 内已经对多时相 token 做了上下文建模。我们早期融合实验沿用官方 probing 接口，在 encoder 输出端对上下文化后的逐月特征求均值；这保留了一部分时序上下文，但 decoder 无法再显式区分月份。方案五正是保留 T 维并在 decoder 中继续建模时间，用来检验这种压缩是否限制了分割性能。

关于 val loss 上升：

> train loss 继续下降而 val loss 回升说明后期出现过拟合或置信度校准恶化，并不表示训练尚未完成。我们已经同时保存最低 val loss 和最高 val mIoU 的模型，并以 val mIoU 早停；最终测试预先固定使用最高 val mIoU checkpoint。

关于物候：

> 仅让大模型自动学习时间规律还不足以证明利用了物候。下一步先从 train 数据统计各作物的月度光谱和植被指数曲线，再通过类别条件时间注意力加入软先验，并与无先验的保留 T 模型做受控对比。

关于可视化：

> 总体 mIoU 不能解释具体错误。后续将同时报告 per-class IoU、混淆矩阵、边界和破碎度指标，并在固定 test 样本上并列展示输入、真值、预测和错误区域。
