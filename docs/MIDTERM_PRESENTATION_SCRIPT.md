## 第 2 页：为什么以 Galileo 为研究基础

### 页面内容

**任务需求与 Galileo 的对应关系**

| 作物制图需要什么 | Galileo 提供什么 |
| --- | --- |
| 利用作物物候变化 | 多时间遥感建模 |
| 同时理解光谱与空间信息 | 多光谱、多模态输入 |
| 减少对像素级标注的依赖 | 大规模自监督预训练 |
| 同时保留类别语义和局部细节 | global + local 预训练目标 |

本项目的基础判断：

> Galileo 已经学习了可迁移的遥感时空表征；本项目研究的不是重新证明预训练有效，而是这些表征能否被更充分地用于像素级作物分割。

### 讲稿

我们首先解释为什么选择 Galileo，并把它视为整个研究的基础。

作物识别不能只看一张影像。不同作物在某一个日期可能有相近的光谱外观，但从播种、生长到收获会形成不同的物候变化。因此，这项任务需要同时处理时间、光谱和空间信息。另一方面，像素级农业标注成本较高，而未标注遥感影像非常丰富，这使自监督预训练具有实际价值。

Galileo 与这些需求具有直接对应关系。它面向多时间、多光谱和多模态遥感数据进行预训练，并通过扩展的掩码建模框架同时学习 global 和 local 特征。全局目标更强调跨区域、跨时间的语义一致性，局部目标则强调较细粒度的空间与光谱信息。

因此，我们不把 Galileo 仅仅当作一个可以替换的 backbone，而把它作为研究前提：它已经通过大规模预训练学习了较丰富的遥感表征。原论文在 PASTIS 上的 probing 结果也说明这些表征具有下游可迁移性。但这只能说明“信息存在”，还不能说明这些信息已经被最有效地转化为像素级结果。这个尚未解决的读取问题，才是我们继续研究 decoder 的起点。

## 第 3 页：为什么选择 DPT

### 页面内容

```text
Galileo encoder 输出：Transformer token
                         ↓
语义分割目标：与原图对齐的像素级类别图
                         ↓
核心缺口：空间重组、边界恢复、跨层融合
                         ↓
DPT：reassemble + projection + fusion + upsampling
```

两个研究问题：

```text
RQ1：空间细化 decoder 是否优于简单线性读取？
RQ2：多层 Galileo 特征是否优于只使用最终层？
```

### 讲稿

在确定 Galileo 是表征基础之后，第二个问题是为什么选择 DPT。

Galileo 的 encoder 输出本质上是一组 Transformer token，而作物语义分割要求输出与原始影像空间对齐的像素类别图。两者之间存在一个明确缺口：token 需要被重新组织为空间特征，低分辨率语义需要逐步上采样，同时还要尽可能恢复农田地块的局部边界。

如果只在最终层特征后接一个简单线性分类头，它可以检验特征是否包含类别信息，但空间恢复能力有限。DPT 正是为 Transformer 的密集预测设计的：它从一个或多个深度取得 token，将其重组为空间特征，再经过投影、融合和渐进上采样生成高分辨率预测。

这与 Galileo 的 global 和 local 表征形成了我们的核心联系。深层特征可能包含更强的全局类别语义，中间层可能保留更多局部空间信息；DPT 提供了一种检验这些层级信息能否互补的结构。

这里需要提前区分“前置 baseline”和“完整 DPT”。我们已经完成的最终层卷积 decoder，以及多层同分辨率融合 decoder，只分别检验空间细化和跨层信息，并不是 DPT 论文的完整复现。完整的 Galileo-DPT 还需要把四层特征重组为不同空间尺度，再由深到浅逐级融合。它目前正在实现，结果将在训练完成后补入。因此，DPT 是本项目的方法来源和当前正在推进的主要方法，而不是对前两个 baseline 的宽泛命名。

## 第 4 页：由研究假设推出技术路线

### 页面内容

```text
PASTIS 多时相输入
  -> 对齐 Galileo 论文输入协议
  -> frozen Galileo ViT-Base
  -> 正确恢复结构化空间 token
  -> 共享 final / 3 / 6 / 9 / 12 层特征缓存
  -> 最终层卷积 baseline / 同尺度多层 baseline
  -> 多尺度 Galileo-DPT / UPerNet-style
  -> 像素级作物类别图
```

实验原则：**固定输入、固定 encoder、固定训练和评测协议，只改变 decoder。**

### 讲稿

基于前两页的判断，我们得到这条技术路线。

首先，将 PASTIS 多时相影像处理为与 Galileo 论文 probing 协议一致的输入。然后完全冻结 Galileo ViT-Base，提取最终层以及第 3、6、9、12 层特征，并在恢复正确空间语义后统一缓存。最后，让不同 decoder 读取同一份缓存，输出像素级作物分类结果。

我们这样设计，是为了建立受控实验。研究变量只放在 decoder 上，输入、encoder、loss、数据划分和评价方法保持一致。这样，两个 baseline、多尺度 DPT 以及 UPerNet 的差异才可以解释为解码方式差异，而不是输入处理或 encoder 训练造成的混杂。

共享缓存还有工程意义：Galileo 只需运行一次，后续 decoder 可以在有限显存设备上独立训练。但缓存首先服务于科研控制变量，节省显存只是附带收益。

## 第 5 页：数据集、任务与研究目标

### 页面内容

**PASTIS 多时相作物语义分割**

```text
输入：Sentinel-2 多时间、多光谱影像
输出：每个像素的背景或作物类别
评价：19 类 mIoU 与 per-class IoU
```

- 2433 个 `128×128` 空间 patch
- 10 个 Sentinel-2 波段
- 18 种作物 + 背景，共 19 个有效类别
- 官方 5-fold 划分；void 像素不参与训练和评价

研究目标：在统一 PASTIS 协议下，比较不同 decoder 对冻结 Galileo 表征的读取能力。

### 讲稿

技术路线明确之后，再说明我们具体在哪个任务上验证。

PASTIS 是农业卫星影像时间序列数据集，每个空间位置具有多次 Sentinel-2 观测，可以反映作物的物候变化。它同时提供像素级作物标签，因此适合检验 Galileo 的多时相表征能否支持密集预测，而不只是图像级分类。

数据集包含 2433 个 128×128 patch、10 个 Sentinel-2 波段，以及 18 种作物和一个背景类，共 19 个有效类别。原始 label 19 是 void，代表类别体系外或边界不可靠的像素，训练与评价时必须忽略。

我们的主要指标是 mIoU：分别计算每个有效类别的交并比，再进行平均。它比总体准确率更能反映少数作物类别是否被有效识别。这里需要明确，我们的目标不是把不同论文中协议不一致的数字直接排名，而是在统一输入、划分和训练设置下，比较 decoder 对同一份 Galileo 表征的读取能力。

## 第 6 页：输入协议与关键工程修正

### 页面内容

```text
原始：[T, 10, 128, 128]
  -> 2018-10 至 2019-09 月度聚合
月度：[12, 10, 128, 128]
  -> 切成 4 个不重叠子块
输入：[12, 10, 64, 64]
  -> Galileo patch size = 4
特征：[768, 16, 16]
```

固定设置：

- train：fold 1/2/3；val：fold 4；test：fold 5
- 官方 `norm_no_clip`，`std_multiplier=2.0`
- 19 个有效类别，void 映射为 `-1`
- Galileo encoder 完全冻结

### 讲稿

项目早期性能偏低的一个重要原因，是输入协议和 token 重组方式没有充分对齐 Galileo 的官方实现。现在我们已经完成了系统修正。

原始不规则时序先聚合为一个完整作物年的 12 个月，同月多个观测取均值，个别缺失月份使用相邻月插值。之后每张 `128×128` 图切为四个 `64×64` 子块，Galileo patch size 设置为 4，因此 encoder 输出 `16×16` 空间网格。

另一个关键问题是 Galileo token 不只是空间 token，还包含时间和通道组。不能把完整 token 序列直接按 256 个空间位置分块平均。当前实现按照 `[H, W, T, channel group]` 的结构进行聚合，先正确对应空间位置，再得到二维特征图。

这些修正保证了后面的 decoder 比较建立在一致、可解释的输入基础上。

## 第 7 页：从 baseline 到完整 DPT 的 decoder 路线

### 页面内容

| 路线 | 输入特征 | 核心目的 | 当前状态 |
| --- | --- | --- | --- |
| 最终层卷积 decoder | 最终层 `16×16` | 检验空间细化是否优于简单读取 | 已完成 |
| 多层同尺度融合 decoder | 第 3/6/9/12 层，均为 `16×16` | 检验跨层信息是否有初步价值 | 已完成首轮 |
| 多尺度 Galileo-DPT | 第 3/6/9/12 层，重组为多尺度 | 检验 reassemble 与渐进融合 | 正在实现，结果待补 |
| UPerNet-style | 第 3/6/9/12 层 | 比较 PPM + FPN 式层级融合 | 已实现，待完整训练 |

### 讲稿

目前的 decoder 研究按照“简单控制实验、完整方法、替代方法”三个层次推进。

第一个 baseline 只读取 Galileo 最终层特征，经过投影、残差卷积和一次上采样得到分割图。它准确的名称是最终层卷积 decoder，用来检验空间细化是否比简单线性读取更有效。

第二个 baseline 同时读取第 3、6、9、12 层，但四层特征都是 `16×16`，经过独立投影后由深到浅相加。它检验多层特征是否存在初步互补，但没有构造多分辨率金字塔，因此不能称为完整 DPT。

正在实现的多尺度 Galileo-DPT 才是本项目对 DPT 论文思想的正式适配。它计划把四层 `16×16` 特征分别重组为 `64×64、32×32、16×16、8×8`，然后从深层到浅层逐级融合。该行的数值和结论暂时留空，训练完成后再填写。

第三条是 UPerNet-style，使用 PPM 和 FPN 融合相同的四层 Galileo 特征。因为这些 Transformer 层的空间网格都是 `16×16`，所以这里融合的是语义深度层级，而不是标准 CNN 中不同空间分辨率的金字塔。

## 第 8 页：实验设置

### 页面内容

| 项目 | 设置 |
| --- | --- |
| Encoder | Galileo ViT-Base，冻结 |
| Loss | Cross Entropy + 0.5 × Dice |
| Optimizer | Prodigy |
| Batch size | 16 |
| Epochs | 100 |
| Seed | 42 |
| GPU | RTX 4060 Laptop，8GB |
| Checkpoint | 最低 validation loss |

模型规模：

- 最终层卷积 baseline：4.92M 可训练参数
- 多层同尺度 baseline：9.06M 可训练参数
- 多尺度 Galileo-DPT：待实现完成后统计

### 讲稿

当前两个 baseline 使用相同的数据、缓存、loss、优化器和 batch size。正在实现的多尺度 DPT 也必须沿用这套协议。Galileo encoder 不参与更新，训练的只有 decoder。

最终层卷积 decoder 约 492 万可训练参数，多层同尺度 decoder 约 906 万，是前者的 1.84 倍。因此当前结果不仅包含融合结构差异，也可能包含模型容量差异，后续需要补参数量匹配实验。多尺度 DPT 的参数量将在实现固定后补入。

本轮训练使用一个 seed，epoch 上限为 100，按照最低 validation loss 保存 checkpoint。这个设置后来暴露出一个问题：语义分割主要指标是 mIoU，而最低 val loss 与最高 val mIoU 不一定出现在同一 epoch。

## 第 9 页：阶段结果与性能定位

### 页面内容

| 方法 | Test loss | Test mIoU | 相对论文 39.2% |
| --- | ---: | ---: | ---: |
| Galileo ViT-Base linear probe，论文参考 | - | 39.20% | - |
| 最终层卷积 decoder | 1.03058 | **45.78%** | +6.58 pp |
| 多层同尺度融合 decoder | **1.00184** | **46.32%** | +7.12 pp |
| 多尺度 Galileo-DPT | 待补 | 待补 | 待补 |

阶段观察：

```text
卷积空间 decoder 方案有效，但 46.32% 不是高水平结果
同尺度多层融合相对最终层 baseline 仅提升 0.54 个百分点
完整多尺度 DPT 尚无结果，当前不能评价 DPT 本身的效果
```

性能背景：PASTIS/U-TAE 原论文报告 `63.1% mIoU`，但其原始不规则时序、端到端训练和完整五折协议与本项目不同，只用于说明当前结果仍有明显提升空间，不能作严格横向排名。

### 讲稿

目前 fold5 test 的结果如下。Galileo 论文在对应 PASTIS 表格中报告 ViT-Base linear probe 为 39.2% mIoU。最终层卷积 decoder 达到 45.78%，多层同尺度融合 decoder 达到 46.32%。这些结果说明更强的空间读取方案超过了论文中的简单线性读取，但从 PASTIS 任务的绝对水平看，46.32% 并不是一个高结果。

PASTIS/U-TAE 原论文在其自身协议下报告 63.1% mIoU。由于它使用原始不规则时序、端到端训练和完整五折交叉验证，不能与我们的月度输入、冻结 Galileo 和单轮 fold 实验直接比较；但这个结果至少说明，当前方案距离成熟的任务专用方法仍有较大提升空间。因此中期阶段的结论不是“我们已经获得高性能”，而是“我们建立了可信实验链路，并验证了 decoder 改造具有初步价值”。

同尺度多层模型相对最终层 baseline 提高 0.54 个百分点，同时 test loss 也有所下降，方向上支持中间层特征可能具有补充价值。但 39.2% 是论文 linear probe，而我们的卷积 decoder 具有更多参数，因此 RQ1 当前属于完整下游方案比较，不是同容量下的严格 decoder 消融。RQ2 的 0.54 个百分点也只来自一个 seed，而且多层模型参数更多，目前只能称为阶段性正向信号，不能判断统计稳定性。

多尺度 Galileo-DPT 的数据和结论目前留空。只有完成 `Reassemble + 多尺度渐进融合` 并在相同协议下训练后，我们才能回答 DPT 是否比两个 baseline 更有效。

## 第 10 页：训练过程中的发现

### 页面内容

| 模型 | 最低 val loss epoch | 最高 val mIoU | 最高 mIoU epoch | epoch 100 mIoU |
| --- | ---: | ---: | ---: | ---: |
| 最终层卷积 baseline | 25 | 47.15% | 47 | 44.16% |
| 多层同尺度 baseline | 11 | 49.34% | 22 | 42.54% |

结论：

- 100 epochs 明显过长
- 多层同尺度 baseline 过拟合更快
- checkpoint 规则与主指标错位

### 讲稿

训练曲线揭示了比最终分数更重要的问题。单层模型在 epoch 25 达到最低 val loss，在 epoch 47 达到最高 val mIoU；多层模型在 epoch 11 达到最低 val loss，却在 epoch 22 达到最高 val mIoU 49.34%。

由于当前只保存最低 val loss，多层模型在 epoch 22 的权重没有保留，无法进行 test 评估。到 epoch 100 时，两个模型训练损失仍在下降，但验证 mIoU 明显回落，多层模型下降更严重。

因此后续必须同时保存 `best_val_loss` 和 `best_val_mIoU`，并在训练前明确最终使用哪一种规则。同时将训练上限缩短，加入基于 val mIoU 的 early stopping。

## 第 11 页：当前成果与局限

### 页面内容

已完成：

- Galileo 官方 PASTIS 输入协议对齐
- Galileo token 的正确空间聚合
- 冻结特征共享缓存与完整训练/评估链路
- 最终层卷积与多层同尺度融合 baseline 首轮 test
- 多尺度 Galileo-DPT 正在由组员实现，结果待补
- UPerNet-style decoder 实现
- TensorBoard、逐类 IoU 与定性可视化工具

当前局限：

- 当前最高 test mIoU 为 46.32%，绝对性能仍不高
- 当前 46.32% 来自同分辨率投影与相加，不能作为完整 DPT 的结果
- 仅一个 seed
- checkpoint 未按 mIoU 保存
- 多层同尺度 baseline 参数更多，尚未做容量控制
- UPerNet 尚无完整结果
- fold5 已用于本阶段汇报，不应继续据此反复调参

### 讲稿

本阶段的主要成果不仅是得到两个 mIoU 数字，更重要的是完成了从原始 PASTIS、Galileo 输入对齐、特征缓存、decoder 训练到 fold5 测试的完整可复现链路。

同时，我们已经实现第三条 UPerNet-style 路线，并补充了 TensorBoard、逐类 IoU 和预测可视化工具。

但当前证据仍然有限。首先，当前最高 test mIoU 为 46.32%，从 PASTIS 作物分割任务看仍不高，说明冻结特征、固定时间聚合和现有 decoder 的组合还有明显瓶颈。这个结果来自多层同尺度融合 baseline：第 3、6、9、12 层经过同分辨率投影后相加，没有构造 DPT 原论文中的多分辨率渐进融合。因此当前结果不能被表述为 DPT 的结果，也不能代表多尺度 DPT 的上限。

其次，实验只有一个 seed，多层模型参数更多，而且 checkpoint 保存规则与 mIoU 主指标存在错位。特别是 fold5 已经用于本阶段汇报，后续不能根据 test 结果继续反复调整超参数，否则会造成测试集信息泄漏。下一轮规则应在训练前固定，主要依靠 fold4 完成选择。

## 可能被问到的问题

### 1. 46.32% 是否算一个高结果？

不算。它高于 Galileo 论文的 39.2% linear probing 参考，但仍低于一些任务专用方法在各自协议下报告的结果。由于协议不同，不能直接进行排行榜式比较。中期阶段的价值在于完成可信实验链路并得到可检验的 decoder 假设，而不是宣称达到 SOTA。下一阶段要通过多 seed、UPerNet、参数量控制和瓶颈消融继续提高证据强度与性能。

### 2. 为什么不直接和 U-TAE 的 63.1% 比？

U-TAE 的 63.1% 来自 PASTIS 原始不规则时序、端到端训练和完整五折交叉验证；我们使用 Galileo 论文的月度 12 时相、`64×64` 子块、冻结 encoder 和单轮 fold 设置。输入和评测协议不同，因此只能作为任务背景，不能直接判断谁优谁劣。

### 3. 为什么假设 Galileo 已经包含可迁移信息？

这是基于 Galileo 的预训练目标和论文下游结果提出的研究假设。最终层卷积 decoder 达到 45.78%，说明最终冻结特征确实具有较强可迁移性，但“是否足够”仍要通过后续多尺度 DPT、UPerNet 和多 seed 实验判断。

### 4. 多层同尺度融合只提升 0.54%，值得吗？

目前只能说方向上有正收益。因为只有一个 seed，而且参数量增加约 84%，不能确认提升来自稳定的多层互补。后续多 seed 和参数量匹配正是为了回答这个问题。

### 5. 当前多层方法是不是完整 DPT？

不是。当前 46.32% 来自四层同为 `16×16` 的特征经过独立投影后逐层相加，它只是跨层信息的控制实验。组员正在实现的多尺度 Galileo-DPT 会增加多分辨率 reassemble 和深到浅渐进融合；其数据和结论尚未产生，不能用当前 46.32% 代替。

### 6. 为什么使用 Prodigy？

Prodigy 能根据梯度自适应估计步长，减少人工学习率搜索。本轮按照组内建议使用 `lr=1`、decoupled weight decay 0.1。但正式实验仍需固定设置并通过多 seed 检验稳定性。

### 7. 为什么冻结 Galileo？

冻结 encoder 可以降低显存和训练成本，也能把研究变量集中在 decoder 上。如果同时微调 encoder，就难以判断提升来自预训练表征调整还是 decoder 设计。

### 8. 为什么前两个 baseline 的旧配置名里仍有 DPT？

它们最初是按 DPT-style 探索命名的，配置文件名为了兼容现有 checkpoint 和运行记录暂不改动，但科研表述已经校正。`single_layer_dpt` 实际是最终层卷积 decoder，`multi_layer_dpt` 实际是多层同尺度融合 decoder；只有包含多尺度 reassemble 和渐进融合的新实现才称为多尺度 Galileo-DPT。

### 9. 为什么当前使用 19 类而不是 20 类？

PASTIS 有背景加 18 种有效作物，共 19 个需要预测的类别。原始编号 19 是 void，不是有效类别，应映射为 ignore index 并从 loss 和 mIoU 中排除。
