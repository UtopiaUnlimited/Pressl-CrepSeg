# 后续研究规划：从通用物候旁路转向困难类别针对性优化

日期：2026-07-16

> **定位更新（2026-07-17）：** 根据助教对后半阶段目标的进一步指导，本文件降为“困难类别诊断与针对性优化”支线，不再作为项目总规划。当前总规划见 [NEXT_STAGE_HETEROGENEOUS_PRIOR_INJECTION_PLAN_2026-07-17.md](NEXT_STAGE_HETEROGENEOUS_PRIOR_INJECTION_PLAN_2026-07-17.md)。现有分类别诊断、类别均衡和多原型建议仍可作为物候任务的个性化分析使用。

任务：基于 frozen Galileo 多时相特征的 PASTIS 作物语义分割

## 1. 当前判断

现阶段不应简单得出“先验无效”的结论，但也不应继续把固定的类别—月份物候表扩展到所有 decoder。当前分类别结果显示，模型误差来源与固定物候先验能够提供的信息并不完全匹配：

- 软质冬小麦、玉米、冬油菜、向日葵、大豆等季节性明确的作物，本身已经能从完整月序列和时间融合中获得较好的识别结果，固定物候表的附加空间有限。
- `Fruits, vegetables, flowers`、`Leguminous fodder`、`Mixed cereal` 等聚合或混合类别内部包含多种作物和管理周期，单一类别曲线难以代表其真实分布。
- `Meadow`、`Grapevine`、`Orchard` 等多年生或长期覆盖类别的关键判别信息不只来自月份变化，还包括空间纹理、地块形态、行列结构和管理方式。
- `Winter triticale`、`Sorghum` 等类别可能同时受到样本不足和近邻类别混淆影响，需要先确认错误是否主要由类别不平衡造成。

因此，后续主线调整为：

> 先诊断每个困难类别的主要错误来源，再选择与错误来源匹配的先验或监督方式；固定外部物候表只保留最小受控实验，不再无条件铺开。

这里保留“先验”这一研究方向，但将先验从统一的日历规则扩展为类别频率、层级语义、类别条件时序原型、空间结构和地块一致性等更有针对性的知识。

## 2. 需要回答的研究问题

后续工作围绕三个问题展开：

1. 当前低 IoU 是由样本不足、类别内部异质、类别间相似，还是空间边界错误造成的？
2. 保留完整时间维并使用可学习时间融合后，固定外部物候知识还能否提供独立信息？
3. 对于固定物候表难以帮助的类别，类别均衡、层级监督、多时序原型或空间先验能否带来更稳定的提升？

对应假设为：

- **H1：类别不平衡假设。** 部分困难类别主要因为有效训练像素和包含该类的 patch 较少而被模型忽略。
- **H2：类内多模态假设。** 聚合类和混合类不能由单一时间曲线描述，需要一个类别包含多个时序原型。
- **H3：信息类型错配假设。** 多年生类别更依赖空间纹理和地块结构，而不是固定的月份权重。
- **H4：物候冗余假设。** 对季节性明确的类别，Temporal Readout 或 3D 时间融合已经学到主要周期，外部日历只提供有限增量。

## 3. 先完成当前实验闭环

### 3.1 四种 Temporal Readout 基线

当前正在评估的四种 P0 模型统一使用：

- 同一份 `temporal_v2` test cache；
- 各自的 `best_val_miou.pt`；
- 相同的 fold5 test 协议；
- 不附加 `--phenology-config`。

需要整理的结果包括：

```text
test loss / mIoU / accuracy / macro F1
19 类 per-class IoU
19 类 per-class F1
固定样本预测图
checkpoint 对应的最佳验证 epoch 与 val mIoU
```

四个 test 的作用是补全既定模型比较，不能根据 fold5 结果继续反复调整超参数。后续新方法的选择必须以 fold4 validation 为依据。

### 3.2 固定外部物候的最小消融

当前 P1 只运行了少量 epoch，暂时不能根据早期曲线下结论。建议只完成一组固定 decoder 的 P0/P1 验证比较：

| 实验 | 设置 | 目的 |
| --- | --- | --- |
| P0 | 无物候旁路 | 严格基线 |
| P1 | 正确外部物候表 | 检查是否存在可见增量 |
| P2 | 类别曲线置乱 | 仅在 P1 通过初筛后运行，验证增益是否来自正确类别对应 |

P1 至少运行到验证 mIoU 进入稳定平台，模型选择仍使用 `best_val_miou.pt`。初筛只看 fold4 validation，不提前运行或反复查看 fold5 test。

建议预先固定以下准入标准：

- P1 相对 P0 的最佳 val mIoU 提升至少 `0.5` 个百分点；或
- 预先指定的季节性目标类别平均 IoU 提升至少 `2.0` 个百分点，同时总体 mIoU 不下降超过 `0.3` 个百分点。

若 P1 未达到任一标准，则记录为“固定外部物候在完整时间融合条件下未显示有效增量”，停止扩展到其他 decoder，也不继续投入 P2 和多 seed。若 P1 通过初筛，再补 P2；只有 `P1 > P0` 且 `P1 > P2`，才进入多 seed 验证。

## 4. 第一优先级：建立困难类别诊断

在修改模型之前，先为当前最佳时间模型生成统一诊断报告。仅有 per-class IoU 还不足以决定优化方向，至少需要以下证据：

### 4.1 类别支持度

统计 train folds `1/2/3` 中每类的：

- 有效像素数与占比；
- 包含该类别的 patch 数；
- 包含该类别的原始地块数；
- 每月有效观测数量；
- 类别在不同 fold 和区域中的分布。

如果低 IoU 与支持度明显相关，优先尝试类别均衡；如果支持度充足但仍低，则转向类内异质或类别混淆。

### 4.2 混淆矩阵

同时输出原始计数、按真实类别归一化和按预测类别归一化的混淆矩阵，用于区分：

- **低 recall：** 模型很少预测该类，可能是类别不平衡或特征不足；
- **低 precision：** 其他类别大量被误判为该类，需要检查相似类别；
- **同类谷物之间混淆：** 适合更细的类别条件时间建模；
- **聚合类与单一作物混淆：** 适合层级监督或多原型表示；
- **只在边界附近出错：** 适合边界损失或地块一致性约束。

### 4.3 空间错误图

为固定样本保存 `RGB / Ground Truth / Prediction / Error Map`，并将错误划分为：

- 地块内部整片分类错误；
- 地块边界错误；
- 小地块遗漏；
- 混合类别与单一作物互相混淆；
- 多年生类别空间纹理识别失败。

诊断阶段的交付物应为一个可重复生成的 CSV/JSON 指标文件和一张混淆矩阵图，而不是只保留终端日志。

## 5. 第二优先级：低成本针对性优化

### 5.1 类别均衡损失

先保持 encoder、时间模块和 decoder 完全不变，只修改损失权重。候选方案按以下顺序单独测试：

1. 基于 train 像素频率的 class-balanced CE；
2. 基于 effective number 的平滑类别权重；
3. 对低 recall 类别使用 focal term；
4. 保留现有 Dice，但限制极少数类别权重，避免训练不稳定。

一次实验只改变一种损失设置。重点观察困难类别平均 IoU、总体 mIoU，以及高分常见类别是否被明显牺牲。

### 5.2 类别感知采样

如果困难类别出现在较少 patch 中，可提高包含这些类别的 patch 被采样概率。采样依据只能来自 train folds，不能读取 validation 或 test 标签统计。

该方法主要解决“训练中很少见到某类”的问题，不适合处理类别内部本身高度混合的情况。

### 5.3 层级辅助监督

在 19 类细分类之外增加一个训练期辅助头，将类别粗分为：

```text
Background
Seasonal annual crops
Perennial / persistent vegetation
Aggregated or mixed crop groups
```

粗类别映射需要在实验前固定。推理时仍输出原来的 19 类，辅助头只用于训练表示。该方案的目标是先让模型学会较稳定的上层语义，再区分细粒度类别，尤其适合聚合类、多年生类和相近谷物类。

## 6. 第三优先级：类别条件的多时序原型

若低成本方法不足，再将“物候先验”升级为类别条件的可学习时序原型，而不是继续加强全局月份向量。

建议结构为：

```text
逐像素 12 月 Galileo 特征
  -> 类别证据投影
  -> 与每类 K 个时序原型计算相似度
  -> 类别条件 temporal logits
  -> 与原分割 logits 门控融合
```

核心约束：

- 推理阶段不读取真实类别；
- 每个像素先由图像特征产生类别证据，再与各类别原型比较；
- 一个类别允许多个原型，以表达不同地区、播期、品种和管理方式；
- 原型只使用 train folds 学习，validation 只用于模型选择；
- 融合门控从较小强度开始，避免先验分支压过图像主分支。

建议原型数：

- 稳定季节性类别：`K=1` 或 `K=2`；
- `Meadow`、`Fruits, vegetables, flowers`、`Leguminous fodder`、`Mixed cereal`：`K=3` 或 `K=4`；
- 多年生类别允许时间分支低权重，并结合空间分支。

这一方向仍属于“先验建模”，但先验变成了由数据学习、能够表达类内多模态的类别条件知识。

## 7. 第四优先级：多年生与聚合类别的空间先验

对于 `Grapevine`、`Orchard`、`Meadow` 等类别，优先考虑：

- 更大感受野或多尺度纹理分支；
- 地块内部一致性约束；
- 边界损失；
- 葡萄园或果园的行列纹理特征；
- 对象级或地块级辅助分类。

对于 `Fruits, vegetables, flowers`、`Leguminous fodder`、`Mixed cereal`，需要接受其标签本身具有高类内异质性。合理目标不是强行让单一原型拟合所有样本，而是：

- 使用多个子原型；
- 使用层级分类；
- 分析标签组成和地理分布；
- 报告该类性能上限可能受标签定义限制。

## 8. 推荐实验顺序

| 阶段 | 实验 | 唯一主要变量 | 进入下一阶段的条件 |
| --- | --- | --- | --- |
| E0 | 当前最佳 Temporal Readout / 3D baseline | 无 | 完成指标与错误诊断 |
| E1 | 类别均衡 loss | loss 权重 | 困难类均值提升且总体不明显下降 |
| E2 | 类别感知采样 | train sampler | 低支持度类别 recall 改善 |
| E3 | 层级辅助头 | coarse auxiliary loss | 聚合类或多年生类稳定改善 |
| E4 | 类别条件多时序原型 | prototype branch | 相近季节作物混淆减少 |
| E5 | 空间/地块先验 | spatial regularization | 多年生类和边界错误减少 |

不建议一次叠加 E1–E5。每个模块先在同一基础模型上单变量比较，通过验证集初筛后再考虑组合。

## 9. 统一判定与实验纪律

每项新方法都按以下规则执行：

1. 固定 train/val/test folds、cache、seed、decoder、优化器和 checkpoint 选择规则。
2. 初筛只使用 fold4 validation，不根据 fold5 test 调参。
3. 预先指定该方法针对的类别组，并同时报告总体和目标组指标。
4. 初筛通过后补 seed `43/44`，报告 mean ± std。
5. 正式候选固定后再运行一次 fold5 test。
6. 不以单个类别偶然上升作为成功；必须检查其他类别是否出现系统性下降。

建议的通用初筛标准为：

```text
总体 val mIoU 提升 >= 0.5 pp
或
目标困难类别组平均 IoU 提升 >= 2.0 pp，
且总体 val mIoU 下降 <= 0.3 pp
```

该阈值是项目内部的资源分配标准，不等同于统计显著性。最终结论仍需多 seed 结果支持。

## 11. 预期研究叙事

后续论文或汇报不再把目标表述为“加入物候先验一定提升性能”，而是：

> 系统分析多时相预训练特征在作物分割中的类别差异，并研究不同类型先验应如何与错误来源匹配。实验先检验固定外部物候表在完整时间融合后的增量信息，再针对类别不平衡、聚合标签、多年生空间结构和类内多模态分别设计监督与先验。

即使固定物候表最终没有提升，也可以形成有价值的阶段结论：

> 当 Galileo 特征保留完整月序列并由下游模块学习时间融合时，统一类别—月份规则与模型已有时间信息存在冗余；困难类别的主要瓶颈更可能来自类别不平衡、标签内部异质和空间结构，而不是缺少月份知识。

后续创新点由“有没有先验”转向“什么类别需要什么先验，以及先验如何以可验证的方式进入模型”。
