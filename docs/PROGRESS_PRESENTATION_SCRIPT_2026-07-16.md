## 第 8 页：对原时间读取方式的复核

### 页面内容

```text
多时相输入
  -> Galileo 跨时间上下文化
  -> F [B,L,T,768,16,16]
  -> 固定 mean(T)
  -> G [B,L,768,16,16]
  -> 二维空间 decoder
```

关键限制：decoder 看不到月份身份，也不能针对不同层和不同空间位置学习月份权重。

### 口头讲稿

复核后我们确认，原方案并非完全忽略时间：十二个月共同输入 Galileo，encoder 内部已经发生跨时间 token 交互。

原方案的主要限制位于 encoder 和 decoder 的接口处。旧缓存对上下文化后的逐月特征直接求平均，得到二维特征图。因此时间上下文仍然存在，但月份身份和月份之间的差异被固定压缩了，后续 decoder 无法再学习某块地在某个月更有辨识价值。

这也是本阶段修改缓存和时间读取模块的原因。

### 版式执行

采用左右对照。左侧画 12 个窄列表示逐月上下文化特征，列上方标自然月；中间只放一个清晰的 `mean(T)` 运算符；右侧放单个二维特征平面。页面底部用两列表格写“保留：跨月上下文”“损失：月份身份、差异化权重”。不使用模糊渐变或抽象光效。

## 第 9 页：从固定平均到可学习时间融合

### 页面内容

**路线 A：独立 Temporal Readout**

```text
[B,L,T,D,H,W]
  -> 月份编码 + 特征打分
  -> softmax(T)
  -> [B,L,D,H,W]
  -> 任意二维空间 decoder
```

**路线 B：decoder 内晚期融合**

```text
[B,L,T,D,H,W]
  -> 时空与跨层联合处理
  -> 高分辨率阶段仍保留 T
  -> 最后学习时间聚合
```

### 口头讲稿

我们目前实现了两类时间融合方式。

第一类是独立的 Temporal Readout。它在每个 Galileo 层、每个空间位置上，根据特征和月份编码学习十二个月的权重，再把时间聚合后的二维特征交给原来的卷积、多层融合、UPerNet 或 Adapted DPT。它的优势是能够和空间 decoder 解耦，便于做受控比较。

第二类是 decoder 内部的晚期融合，以 3D-Aware DPT 为当前实现。它在空间重组和跨层融合过程中持续保留时间维，最后才聚合月份。这条路线考察更深度的时空联合建模。

两条路线共同研究 Galileo 逐月输出的下游读取方式，但采用了不同机制，后续实验中将分别记录和比较。

### 版式执行

使用上下两条平行泳道，不使用左右两个独立卡片。上方泳道是 Temporal Readout：在时间轴处汇聚后再进入四种二维 decoder；下方泳道是 decoder 内晚期融合：时间轴贯穿空间融合，直到输出前汇聚。相同的 Galileo 输入节点上下对齐，相同操作使用相同颜色；唯一差异点用黄色底标出。

### 口头讲稿

在讨论新时间路线之前，已有结果已经说明空间 decoder 的设计是必要的。同一训练协议下，Linear Head 为 32.09%，最终层卷积提高到 46.02%，多层同尺度融合进一步达到 48.01%。这说明 Galileo frozen feature 包含可迁移信息，但仅靠线性映射不足以恢复细粒度空间结构。

## 第 13 页：物候先验的构建与不确定性处理

### 页面内容

```text
P_ext[class, calendar_month] in [0,1]
19 个有效类别 × 12 个自然月
```

先验不直接凭经验填写月度权重，而是先建立可审计的原始阶段表：逐类记录资料适用地区、播种、出苗或返青、快速生长、开花、成熟和收获的时间范围，并保存来源、年份、用途或品种以及可信度。

```text
原始农学资料
  -> 类别-地区-生育阶段-时间范围
  -> 统一映射到 12 个自然月
  -> 按地区覆盖比例生成 P_ext
```

在保留当前二维接口时，月份值由固定规则计算，而不是人工指定 `0.65` 或 `0.80`：

```text
P_ext(c,m) = 支持类别 c 在月份 m 处于目标生育阶段的有效地区数
             / 类别 c 在月份 m 具有有效资料的地区总数
```

数值表示来源对“该类别在该月处于目标生育阶段”的一致支持程度，不是类别概率，也不是法国作物的真实物候标签。数值化规则将在训练前冻结。

| 类别类型 | 数量 | 第一版处理 |
| --- | ---: | --- |
| 可对应主要作物 | 13 | 来源支持的宽软曲线 |
| 聚合或混合类别 | 5 | 均匀、宽窗口或后续 `P_data` |
| Background | 1 | 中性先验 |

### 口头讲稿

我们的外部先验最终表示为一张 19×12 的类别-月份软矩阵，但矩阵不是根据感觉直接打分。我们先把农学资料整理成类别、地区、生育阶段和时间范围，再按照不同地区对同一月份的支持比例生成数值。这样每个权重都能够回溯到原始资料和固定公式。

PASTIS 的 19 个有效标签不能简单理解为 19 种植物。其中 13 类能够较直接对应主要作物，另外 5 类是草地、果蔬花卉、豆科饲料、果园和混合谷物等聚合类别，Background 也不是作物。对于聚合类别，我们不强行指定一种代表植物，而是暂时使用中性或宽窗口。

此外，PASTIS 覆盖法国四个区域，品种、播期和气候存在差异，因此地区之间不一致时会自然得到介于 0 和 1 之间的软值，而不是人为选择一个特征日期。聚合类别和证据不足的类别保持中性，不强行构造周期。外部先验还会与仅由 train folds `1/2/3` 统计的 NDVI 月度曲线分别比较，并通过无先验、正确先验和类别置乱先验验证类别-月份对应关系是否真正有效。

本页的核心结论是：先验权重由可追溯的农学时间范围按照预先固定的规则生成，而不是由研究者主观指定数值。当前手工软曲线仅作为接口联调草案，不能作为正式实验中的外部知识结论。

### 版式执行

主视觉使用 `19×12` 条件着色表：行是类别，列是自然月，颜色只表示相对权重。旁边用小表格列出三种映射类型，不在主页面堆放所有来源链接。

## 第 14 页：通用物候旁路的注入方式

### 页面内容

```text
Galileo逐月特征 F[l,t] ─────────────────────┐
                                            ├→ F'[l,t] → 时间模块 → 空间decoder
P_ext[:, month_t] → MLP(19→128→768) → q[t] ┘

F'[l,t] = F[l,t] + alpha * q[t]
```

张量：

```text
F[l,t] : [B,T,768,H,W]
q[t]   : [B,T,768,1,1]
```

### 口头讲稿

旁路位于 frozen Galileo 特征和后续 decoder 之间。对于每个真实自然月，我们取出这个月份对应的完整 19 类物候状态，经过一个轻量 MLP 投影到 Galileo 的 768 维特征空间，再作为残差广播到该月份、各空间位置和所选 Galileo 层。

这里注入的是该月份整体的类别物候背景，而不是先读取某个像素的真实类别，再为它选择月份。模型在推理时不知道真实标签，因此不会产生标签泄漏。每个像素最终属于哪种作物，仍然由遥感特征和 decoder 判断。

之所以称为旁路，是因为 Galileo 图像主分支保持不变，先验从独立元数据分支进入，而且可以通过配置完全关闭。旁路输出仍是统一的 768 维逐月特征，因此后面可以接 Temporal Readout 加任意二维 decoder，也可以接内部晚期融合的 decoder。

### 版式执行

使用单一左到右数据流。Galileo 特征主干位于中轴；物候表从下方进入 MLP，再在 decoder 之前通过一个明确的加号汇入。每个节点下方标张量形状，`768` 维注入位置用黄色描边；decoder 只画一个可替换插槽，下面列出 Temporal Readout 路线和 3D 路线，不分别画多个 decoder。

## 备份 2：第一阶段优先采用软旁路的考虑

硬筛选暂不作为第一阶段主方案，原因是：

1. 推理阶段不知道真实作物类别，不能按真值类别选择月份；
2. 不同区域、天气、品种和播期会造成物候漂移；
3. 删除月份可能丢失收获后残留、冬季覆盖等判别信息；
4. 软旁路允许模型使用、忽略或修正人工先验，更适合作为第一步验证。

后续若软先验有效，可以使用固定 `T=12` 的 temporal mask 做硬/软筛选消融，而不是生成不定长输入。

## 备份 3：现阶段的结论边界

- 不能把 3D-Aware DPT 的全部提升归因于时间融合；
- 不能把 `P_ext` 称为法国作物的真实物候日历；
- 不能把 PASTIS 的 19 个有效类别称为 19 种植物；
- 不能在 P0/P1/P2 完成前声称物候先验已经有效；
- 不能根据 fold5 test 结果反复选择超参数或修改先验；
- 单 seed 结果只能作为阶段证据，不能声称具有统计显著性。

这些限制不是削弱工作，而是明确当前证据能够支持到什么程度。
