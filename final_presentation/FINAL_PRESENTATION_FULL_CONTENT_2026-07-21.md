## 前半部分的核心叙事

前半部分不是按日期汇报做过哪些模型，而是先从农业遥感的现实需求和方法发展中推出项目问题，再回答三个连续问题：

1. 为什么作物制图需要卫星影像时间序列，任务专用时序网络已经解决了什么？
2. 遥感基础模型带来了什么机会，它的冻结表征迁移到像素级任务时还缺少什么？
3. 决定下游性能的主要因素，是空间恢复、时间读取，还是外部知识？

对应逻辑为：

```text
大范围作物制图需要低成本、可重复的时序观测
  -> 任务专用时序网络有效，但依赖具体任务训练
  -> 遥感基础模型提供通用多光谱、时空表征
  -> frozen Galileo 与像素级分割之间仍存在读取缺口
  -> 先校正数据、输入和 token 语义
  -> 比较不同空间读取方式
  -> 发现 decoder 前固定时间平均是主要瓶颈
  -> 从固定平均推进到可学习时间融合
  -> 3D-Aware 将 mIoU 提升到 0.59945
  -> 但困难类别、类别异质性和知识边界仍然存在
  -> 研究问题自然转向异构先验注入
```

## 第 2 页：研究背景——为什么需要时序作物制图

### 页面标题

**农业遥感需要把全年动态转化为地块级作物地图**

### 页面主体

建议使用“应用需求—时序价值—传统瓶颈”三列表格：

| 作物地图为什么重要 | 为什么必须看时序 | 传统路线的瓶颈 |
| --- | --- | --- |
| 产量估算、轮作监测与水资源调度的基础数据层 | Sentinel-2：10 m、约 5 天重访、多光谱连续观测 | 田间调查和专家解译成本高，难以高频更新 |
| 区域尺度需要大范围、周期性、低成本制图 | 不同作物在生长季形成独特的时间谱线 | 单时相和手工植被指数容易忽略物候动态 |
| 最终结果需要与地块边界对齐的像素级类别图 | 全年曲线有助区分物候相近的作物类型 | 深度时序模型仍受像素级标注稀缺限制 |

页面下方给出任务定位：

> 核心矛盾：既要保留全年物候动态与地块边界，又要降低对高成本像素标注的依赖。

### 建议讲稿

> 精准农业首先需要一张可信的作物类型空间分布图。产量估算、轮作监测、水资源调度和农业补贴核查，都要以这张基础数据层为起点。区域尺度上，卫星遥感凭借覆盖广、重复观测和成本低，基本是持续更新作物地图的可行路径。
>
> Sentinel-2 提供 10 米空间分辨率、约 5 天重访周期和多光谱观测，可以把同一地块一整年的变化记录下来。不同作物在播种、生长和收获阶段会形成不同的时间谱线；只看单期影像，即使加入 NDVI 等手工特征，也容易混淆物候相近的类别。与此同时，像素级农业标注需要田间调查或专家解译，获取成本很高。PASTIS 提供了法国 2018—2019 作物年的 Sentinel-2 时序与 18 类作物加背景标签，作为我们的统一验证平台。

### 建议页脚来源

PASTIS / U-TAE；Pixel-Set Encoder + Temporal Self-Attention。

### 过渡

> 要降低对人工标注的依赖，下一步就要看自监督预训练能提供什么，又留下了什么问题。

## 第 3 页：方法演进——为什么引入自监督遥感基础模型

### 页面标题

**标注瓶颈推动遥感方法从任务监督走向自监督基础模型**

### 页面主体

建议使用“传统监督路线—自监督基础模型”两栏对照：

| 传统监督路线与瓶颈 | 自监督预训练与 Galileo |
| --- | --- |
| 单时相 + 植被指数 + RF/SVM：依赖手工特征，忽略完整物候 | 从大规模无标签 Sentinel-2 数据学习通用时空表征 |
| ConvLSTM、TempCNN、Transformer：增强时间建模，但仍依赖任务标注 | ViT 输出多层、多时刻、可迁移的卫星时序 token |
| 农业像素标签需要实地调查或专家解译，规模扩展成本高 | 形成 frozen encoder + 轻量 decoder 的下游迁移范式 |

方法演进带：

```text
传统任务监督模型
       -> 大规模无标签自监督预训练
       -> Galileo 通用时空表征
       -> 下游时序特征解码与异构先验融合
```

### 建议讲稿

> 传统方法最早依靠单时相影像、NDVI 等手工特征，再用随机森林或支持向量机分类；后来 ConvLSTM、TempCNN 和 Transformer 提升了时序建模能力，但仍然围绕具体任务和人工标签训练。农业像素级标注的成本决定了这种路线很难无限扩展。
>
> 自监督学习通过掩码建模或对比学习，从大量无标签数据中先学通用表征，再迁移到下游任务。Galileo 以 ViT 为骨干，在大规模全球 Sentinel-2 数据上进行掩码图像建模，输出多层、多时刻、可迁移的时空 token。这样我们可以冻结 encoder，把有限标注主要用于下游读取。
>
> 但基础模型只解决了表征从哪里来，并没有自动回答这些 token 怎样恢复成高分辨率作物地图，以及物候、气候、土壤和地理等外部知识怎样与视觉特征交互。这正是本项目继续研究的部分。

### 建议页脚来源

Galileo；PASTIS/U-TAE；自监督学习相关工作。

### 过渡

> 基础模型解决了“表征从哪里来”，接下来要把“怎样解码时序特征、怎样融合异构先验”明确成可验证的研究问题。

## 第 4 页：研究内容——围绕空间、时间和先验逐层推进

### 页面标题

**研究问题来自基础模型与像素级任务之间的三个缺口**

### 页面主体

页面上半部分先呈现缺口：

```text
Galileo：输出多层、多时间 Transformer token
                         ↓
PASTIS：要求与地块边界对齐的 19 类像素图
                         ↓
缺口：token 如何恢复空间？月份何时融合？外部知识如何补充？
```

页面下半部分使用三行研究问题：

| 层次 | 核心问题 | 项目中的验证路线 |
| --- | --- | --- |
| RQ1 空间读取 | frozen Galileo token 能否被可靠恢复为像素级结果？ | Linear → 最终层卷积 → 多层融合 |
| RQ2 时间读取 | 固定平均是否过早丢失月份身份？ | Temporal Readout → 3D-Aware |
| RQ3 知识注入 | 强视觉模型之后，异构先验能否提供独立信息？ | Global Add → CA-HPI → SA-SFiLM |

### 建议讲稿

> 有了前面的背景，我们把问题具体化为三个层次。第一，Galileo 输出的是 token，PASTIS 需要的是和地块边界对齐的像素图，中间需要一个可靠的空间读取过程。第二，12 个月虽然一起进入了 encoder，但如果过早固定平均，decoder 可能看不到月份之间的差异，所以还要研究时间维怎么保留、什么时候压缩。第三，等视觉时间信息已经被充分利用之后，物候、气候、土壤和地理知识还能不能提供独立补充。这个顺序很重要：先把视觉表征读好，再讨论外部知识，而不是一开始把所有因素混在一起。

### 过渡

> 为了让这三个问题能够被受控比较，我们需要先固定任务、数据流和评估纪律，并修正早期输入链路中的问题。

## 第 5 页：研究流程与实验基础

### 页面标题

**先建立可信基线，再逐步增加时间结构和外部知识**

### 页面主体

上半部分使用项目总流程：

```text
研究背景与任务定义
  -> 对齐 PASTIS—Galileo 输入协议
  -> 空间读取：Linear / 卷积 / 多层 decoder
  -> 时间读取：Temporal Readout / 3D-Aware
  -> 先验注入：Global Add / CA-HPI / SA-SFiLM
  -> 总体与逐类结果、诊断和适用边界
```

下半部分保留最少的任务和协议事实：

```text
PASTIS Sentinel-2 [T,10,128,128]
  -> 2018-10 至 2019-09，聚合为 12 个月
  -> 每张切为 4 个 64×64 tile
  -> Galileo patch size 4，encoder 完全冻结
  -> 多层、多月份时空特征
  -> 19 类 64×64 segmentation
```

右侧固定协议：

| 项目 | 固定设置 |
| --- | --- |
| 数据划分 | train folds 1/2/3，validation fold4，test fold5 |
| 模型选择 | fold4 最高 `val_mIoU` checkpoint |
| 主指标 | mIoU，并报告 accuracy、macro F1 与逐类结果 |
| 控制变量 | 输入、frozen Galileo 和评估规则固定 |

底部使用一行“问题—修正—作用”：

| 早期问题 | 关键修正 | 科研作用 |
| --- | --- | --- |
| 作物年、token 语义和重复 encoder 运行存在风险 | 对齐月度输入；按空间/时间/通道组恢复 token；建立 `spatial_v1`、`temporal_v2` 缓存 | 避免把协议错误误认为模型能力差，并让不同 decoder 共享同一表征 |

### 建议讲稿

> 具体做法可以概括成一条流程。我们先把 PASTIS 和 Galileo 的输入协议对齐，建立共享的冻结特征缓存；然后先比较空间 decoder，再比较时间读取方式；在 3D-Aware 形成强视觉基线之后，才接入物候、气候、土壤和地理先验。每一步只增加一个新的问题，最后同时看总体指标、逐类变化和模块诊断。这样即使某个先验方案没有带来明显的总体提升，我们也能知道它改变了哪些类别、在哪个环节出现了问题。

> 数据划分和评估规则也在这里固定：train 使用 folds 1/2/3，fold4 选择 checkpoint，fold5 只做最终测试。项目早期结果偏低时，我们先检查了作物年、归一化和 token 的空间—时间—通道语义，并建立了共享缓存。后面所有比较都建立在这条统一链路上。

## 第 6 页：空间解码探索

### 页面标题

**空间细化显著释放了 Galileo 冻结表征**

### 本页目的

用最少的模型和结果说明：冻结表征并不是不可用，关键在于下游如何恢复空间结构。

### 页面主体

建议左侧使用三条简化结构流，右侧放结果表。不要绘制柱状图。

| 方法 | 读取方式 | 可训练参数 | Test mIoU |
| --- | --- | ---: | ---: |
| 同协议 Linear Head | 最终层 token 直接线性分类 | 0.234M | 32.09% |
| 最终层卷积 | 最终层特征 + 卷积细化和上采样 | 4.925M | 46.02% |
| 多层同尺度融合 | layer 3/6/9/12 投影后融合 | 9.058M | **48.01%** |

结果读法：

- 最终层卷积相对 Linear 提升约 `13.92 pp`；
- 多层特征相对最终层卷积继续提升约 `1.99 pp`；
- 说明 Galileo 已包含可迁移信息，但线性可分性不等于像素级可恢复性；
- 参数量不同且目前主要是 seed42，不能把全部提升解释为单一模块的单位参数优势。

### 建议讲稿

> 先看空间读取。线性 head 的 mIoU 是 32.09%，加入卷积细化后到了 46.02%，再把第 3、6、9、12 层一起读进来，达到 48.01%。这个差距说明两件事：Galileo 的冻结特征确实包含作物信息，但把 token 直接做线性分类，还不足以恢复像素级边界；中间层也确实提供了一些补充。当然，这里是完整下游方案的比较，模型参数量和 seed 数并不完全相同，所以我们把它看作空间读取的阶段性证据，而不是某个单独模块的纯消融。

### 过渡

> 空间 decoder 已经带来明显提升，但这些方案仍共享一个没有被单独检验的操作：在进入 decoder 之前固定平均 12 个月。

## 第 7 页：时间瓶颈的发现

### 页面标题

**旧接口保留了时间上下文，却过早丢失月份身份**

### 本页目的

准确解释旧模型“有时间信息但不会自主读取时间”的区别，这是引出时间融合的关键页面。

### 页面主体

使用单条张量流：

```text
12 个月共同进入 Galileo
  -> encoder 内发生跨时间交互
  -> F [B, L, T, D, H, W]
  -> 固定 mean(T)
  -> G [B, L, D, H, W]
  -> 2D decoder
```

右侧放两句解释：

| 已经保留的能力 | 被固定平均限制的能力 |
| --- | --- |
| 每个月 token 已包含 encoder 内的跨时间上下文 | decoder 无法保留月份身份 |
| Galileo 已看到完整 12 个月序列 | 不同层、空间位置只能使用同一种平均方式 |

### 建议讲稿

> 这里有一个容易混淆的地方：旧方案不是没有时间信息，而是把时间信息压缩得太早。12 个月会一起进入 Galileo，所以每个月的 token 已经互相看过了；问题在于到了 decoder 前，我们直接把 12 个月平均掉。这样 decoder 只看到一张二维特征图，不能再根据位置、层次和作物类型判断哪个月份更重要。于是我们接下来要问的就是，时间维应该保留到哪里，最后又该怎样压缩。

### 过渡

> 我们围绕“何时压缩时间维”设计了两条递进路线。

## 第 8 页：两种可学习时间融合

### 页面标题

**从固定时间平均推进到可学习时间融合**

### 本页目的

清楚区分 Temporal Readout 与 3D-Aware，同时说明它们回答的是同一个科研问题。

### 页面主体

使用上下两条平行流程，配合比较表：

| 路线 | 核心过程 | 时间维保留到哪里 | 研究价值 |
| --- | --- | --- | --- |
| Temporal Readout | 每层、每个空间位置学习 12 个月权重，再送入 2D decoder | decoder 之前 | 作为通用时间读取接口，可接 Single、Multi、DPT、UPerNet |
| 3D-Aware DPT | 多层时空重组、attention 与逐级融合，全程保留 T，最后 query pooling | 高分辨率融合之后 | 检验更晚压缩时间维是否更有利 |

两条结构流可写为：

```text
Temporal Readout：F[B,L,T,D,H,W] -> learned temporal weights -> 2D feature pyramid -> decoder

3D-Aware：F[B,L,T,D,H,W] -> 3D reassemble/fusion -> [B,T,C,64,64] -> query pooling -> logits
```

### 建议讲稿

> 我们做了两种尝试。Temporal Readout 放在 decoder 前，让每一层、每个空间位置自己学习 12 个月的权重，后面的二维 decoder 不用改，所以它比较通用。3D-Aware 则更进一步，把时间维带进多层重组和高分辨率空间融合，最后才用查询池化得到分割结果。它们的区别可以简单理解为：一个学习“怎么读月份”，另一个把时间保留得更久再做空间融合。两条路线其实都在回答同一个问题——时间是不是应该晚一点再压缩。

### 过渡

> 实验结果表明，可学习时间读取在多种二维 decoder 上都有效，而更晚融合的 3D-Aware 取得了当前最强结果。

## 第 9 页：时间融合结果

### 页面标题

**保留时间结构后，Test mIoU 提升到约 60%**

### 本页目的

呈现项目最明确的性能成果，并将 3D-Aware 固定为后续先验研究的强基线。

### 页面主体

表一：Temporal Readout 适配不同二维 decoder 的 fold5 结果。

| 2D decoder + Temporal Readout | Test mIoU | Test accuracy | Test F1 |
| --- | ---: | ---: | ---: |
| Single | 0.54938 | 0.80589 | 0.68833 |
| Multi | **0.56154** | **0.81030** | **0.69712** |
| Galileo-Adapted DPT | 0.54637 | 0.80904 | 0.68433 |
| UPerNet | 0.54442 | 0.80342 | 0.68279 |

表二：最终 3D-Aware 基线。

| 方法 | Test loss | Test mIoU | Test accuracy | Test F1 |
| --- | ---: | ---: | ---: | ---: |
| 3D-Aware，无先验 | 1.03279 | **0.59945** | 0.81781 | 0.73133 |

页面下方只放两个结论：

1. Temporal Readout 证明可学习月份读取可以跨多种二维 decoder 使用；
2. 3D-Aware 进一步把时间维保留到高分辨率融合之后，取得项目当前最强基线。

### 建议讲稿

> 先看 Temporal Readout。它接到四种二维 decoder 上以后，mIoU 都在 0.54 到 0.56 之间，Multi 最高，为 0.56154。再看 3D-Aware，mIoU 达到 0.59945，macro F1 是 0.73133。这是项目目前最明确的一次提升，也说明时间读取方式确实是关键变量。后面的先验实验就固定在这个 3D-Aware 基线上，这样先验带来的变化不会被较弱的视觉模型掩盖或放大。

### 过渡

> 但是约 60% 的总体 mIoU 并不意味着所有类别都已经解决。逐类结果中仍然存在明显的类别异质性。

## 第 11 页：V1——固定物候广播与反事实检验

### 页面标题

**先验证物候内容是否有信号，再讨论更复杂的注入方式**

### 页面目的

V1 不是后面方法的最终方案，而是一个必要的最小实验：先判断模型是否能从正确的类别—月份物候表中获得区别于随机置乱的信号。

### 页面主体

左侧展示 Global Add 数据流：

```text
类别—月份物候表 [19, 12]
        -> MLP / embedding
        -> 先验残差
        -> 广播到所有空间位置
        -> 加到 Single Temporal Readout 特征
```

右侧使用结果表：

| 实验 | 物候输入 | Test mIoU | 相对 P0 |
| --- | --- | ---: | ---: |
| P0 | 无先验 | 0.54938 | — |
| P1 | 正确类别对应的物候 | **0.55316** | **+0.378 pp** |
| P2 | 类别—物候对应关系置乱 | 0.54784 | −0.154 pp |

页面下方写清实验身份：P0/P1/P2 都是 Single Temporal Readout 组，不能与 3D-Aware B0 或后面的 CA-HPI/E10 混为同一组。

### 建议讲稿

> 我们先从最直接的方案开始。把类别—月份物候表经过一个小编码器，得到先验残差，再广播到所有空间位置。这个方法很简单，但它能回答一个基本问题：正确的物候内容，是否比没有物候、或者把类别对应关系打乱，更有用。
>
> 结果是 P0 的 mIoU 为 0.54938，正确物候 P1 是 0.55316，提升 0.378 个百分点；把类别对应关系打乱以后，P2 降到 0.54784。这个差异说明物候内容可能确实有一点信号，但提升很小，而且广播方式太粗：同一份类别先验被无差别地送到所有空间位置。它更适合作为方向验证，不适合作为最终的通用注入方案。

### 过渡

> 所以下一步不是继续修改这张物候表，而是把“先验”抽象成一个可以接收不同来源、并允许视觉内容选择的接口。

## 第 12 页：统一异构先验接口

### 页面标题

**先验来源不同，但可以通过同一个接口进入视觉网络**

### 页面目的

说明项目的通用性来自哪里：不是把所有先验都假装成同一种数据，而是保留来源、置信度和有效性信息，再交给后续融合模块处理。

### 页面主体

| 来源 | 数据形态 | 先验粒度 | 接口中的辅助信息 |
| --- | --- | --- | --- |
| M1 物候 | 类别×月份强度与置信度 | 19×12，编码为 228 个 token | confidence、unknown/mask |
| M2 气候 | 月度温度、降水、辐射、表层水分 | 12 个月 | source id、valid mask |
| M3 土壤 | 多深度土壤属性及不确定性 | 3 个深度/属性组 | confidence、source id |
| M4 地理 | train-fold 标准化经纬度 | 1 个静态上下文 token | source id、空间边界说明 |

统一形式：

```text
PriorBatch = {
  tokens,
  mask,
  confidence,
  source_id / type_id
}
```

底部保留三条边界：

- M2/M3/M4 是外部背景或模型化数值，不称为地块实测数据；
- 统计、标准化和映射只使用允许的外部资料或 train folds；
- 先验接口不读取真实像素类别，不用 test label 选择先验。

### 建议讲稿

> V1 的问题不只是物候表本身，而是它默认所有先验都能用同一种方式广播。我们因此先把先验数据整理成统一的 PriorBatch。M1 是类别和月份的物候信息，M2 是月度气候背景，M3 是土壤属性，M4 是地理上下文。它们的来源、粒度、token 数量和置信度都不同，所以接口不会把这些差异抹掉，而是把 token、有效性 mask、置信度和 source id 一起传给融合模块。
>
> 这样做的意义是，后面的模型可以学习“当前视觉内容应该参考哪个来源、哪些 token 可以使用”，而不是在输入阶段把所有数值简单拼接。我们也明确了数据边界：这些信息是外部背景或模型化资料，不把它们表述成地块实测数据，更不使用 test 标签挑选先验。

### 过渡

> 有了统一接口，第一种通用注入架构就是让每个视觉 token 主动查询整个先验库。

## 第 13 页：V2——CA-HPI 内容感知先验注入

### 页面标题

**CA-HPI：让每个视觉时空位置主动查询先验库**

### 页面目的

必须把 CA-HPI 讲成一条明确的数据流，重点说明它和 V1 的区别：V1 是固定广播，CA-HPI 是视觉内容驱动的查询。

### 页面主体

建议只讲一个视觉 token，避免把整个网络画得过于复杂：

```text
视觉时空 token V_l
       -> Query Q

PriorBatch tokens Z
       -> Key K / Value V
       -> mask + confidence + source balance

Q 与 K 做 cross-attention
       -> 候选先验上下文 Δ_l
       -> content gate g_l
       -> strength α_l
       -> 写回 decoder 前视觉特征
```

数学关系只放三行：

```text
Q = W_q LN(V_l),      K,V = W_k,W_v LN(Z)
Δ_l = Attention(Q, K, V; mask, confidence)
V'_l = V_l + tanh(α_l) · g_l · Δ_l
```

| V1 Global Add | V2 CA-HPI |
| --- | --- |
| 先验表编码后广播到所有位置 | 每个视觉位置有自己的 Query |
| 不区分当前位置是否需要该知识 | 根据视觉内容选择相关先验 token |
| 缺失/低置信度只能粗略处理 | mask、confidence 和 gate 共同控制 |
| 先验作用方式固定为加法 | 逐层 strength 控制残差幅度 |

### 建议讲稿

> CA-HPI 的核心变化是把先验从“广播信号”变成“可查询的知识库”。我们还是从同一个视觉 token 讲起：视觉 token 经过投影得到 Query，所有先验 token 经过 Key 和 Value 投影。Query 和先验库做 cross-attention，每一个空间位置、月份和层都可以得到不同的先验上下文。
>
> 这里还会同时使用 mask 和 confidence。无效或缺失的先验不会参与注意力，低置信度信息也不会和可靠信息完全等价。得到候选残差之后，再用 content gate 决定当前位置接不接受，最后通过逐层 strength 写回 decoder 前的视觉特征。输出的形状不变，decoder 本身不需要知道先验来自哪里。
>
> 所以它和 V1 的区别不是“多加了一个 attention”。V1 默认同一份信息对所有位置都适用，CA-HPI 则把选择权交给当前视觉内容。这是从固定先验到内容感知先验的一步。

### 过渡

> 但把所有来源放进一次 attention 后，模型是否真的能公平地比较不同来源，需要用多源结果来检验。

## 第 14 页：V2 结果——多源先验不是越多越好

### 页面标题

**M1 有弱正信号，但多源直接竞争会带来类别冲突**

### 页面主体

总体结果统一以 3D-Aware B0 为参照：

| 实验 | 先验来源 | Test mIoU | 相对 B0 |
| --- | --- | ---: | ---: |
| B0 | 无先验 | 0.59945 | — |
| E7 | CA-HPI + M1 | **0.60443** | **+0.498 pp** |
| E8 | CA-HPI + M1/M2/M3 | 0.60192 | +0.247 pp |
| E9 | CA-HPI + M1/M2/M3/M4 | 0.59395 | **−0.550 pp** |

代表性类别变化：

| 类别 | E7 M1 相对 B0 | E9 M1234 相对 B0 | 说明 |
| --- | ---: | ---: | --- |
| Grapevine | +4.82 pp | +2.28 pp | 物候/环境知识可能有帮助，但多源后收益减弱 |
| Orchard | +0.67 pp | **+4.01 pp** | 地理或环境 context 可能提供补充 |
| Sorghum | **+3.08 pp** | **−9.25 pp** | 对来源组合高度敏感 |
| Leguminous fodder | −0.69 pp | −4.10 pp | 多源组合未改善困难类别 |

页面结论：总体 mIoU 的变化来自类别正负变化的抵消，不能简单说“加来源有效”或“加来源无效”。

### 建议讲稿

> 结果首先支持了 CA-HPI 的基本方向。只使用 M1 时，E7 的 mIoU 是 0.60443，比 B0 高 0.498 个百分点；加入 M2 和 M3 后，E8 仍然略高于 B0，但降到 0.60192；四个来源全部放进 CA-HPI 后，E9 反而降到 0.59395。
>
> 逐类结果更能说明问题。Grapevine 在 M1-only 下提高 4.82 个百分点，说明某些类别确实可以利用物候知识；Orchard 在四源组合下提高 4.01 个百分点，说明地理或环境背景可能提供补充。但 Sorghum 从 M1-only 的提升 3.08 个百分点，变成四源组合下降 9.25 个百分点。也就是说，多源先验不是来源越多越好，所有 token 放在一次 attention 里竞争时，不同来源可能会互相干扰。
>
> 这组结果给 V3 提出了一个很具体的要求：先在来源内部选择，再比较来源之间的权重，最后还要控制先验到底改动哪些视觉通道和空间位置。

### 过渡

> V3 SA-SFiLM 就是针对这个问题做的结构重设计。

## 第 15 页：V3——SA-SFiLM 与 E10 结果

### 页面标题

**SA-SFiLM：先选来源，再决定通道和空间位置**

### 页面主体

建议把架构和结果放在同一页，但架构占上半部分，结果占下半部分。

架构数据流：

```text
视觉 token V_l
   ├─ 在 M1 内部 attention -> context_1 ┐
   ├─ 在 M2 内部 attention -> context_2 ├─ Source Gate -> context
   ├─ 在 M3 内部 attention -> context_3 ┤                    ├─ FiLM γ/β
   └─ 在 M4 内部 attention -> context_4 ┘                    └─ Spatial Gate
                                                               ↓
                                  decoder 前残差写回 V'_l
```

| 步骤 | 解决的问题 |
| --- | --- |
| 来源内 attention | 防止 token 数较多的来源直接压过其他来源 |
| Source Gate | 根据当前视觉内容决定 M1–M4 的相对权重 |
| FiLM `γ/β` | 明确控制哪些视觉通道缩放、哪些通道平移 |
| Spatial Gate | 决定当前空间位置和月份接受多少调制 |
| Residual strength | 保持输出形状不变，并控制逐层注入幅度 |

严格架构对照：E9 和 E10 使用相同的 3D-Aware、M1–M4、训练协议和 decoder 前插入点，只改变 CA-HPI/SA-SFiLM 融合方式。

结果表：

| 实验 | Test loss | Test mIoU | 相对 B0 | 相对 E9 | Test F1 |
| --- | ---: | ---: | ---: | ---: | ---: |
| B0：无先验 | 1.03279 | 0.59945 | — | — | 0.73133 |
| E9：CA-HPI M1–M4 | 1.06063 | 0.59395 | −0.550 pp | — | 0.72509 |
| E10：SA-SFiLM M1–M4 | 1.22321 | **0.60073** | **+0.128 pp** | **+0.678 pp** | 0.73051 |

### 建议讲稿

> V3 不是简单在 CA-HPI 后面再加一个 FiLM。它把先验选择拆成了几个层次。首先，每个来源内部单独做 attention，得到四份 context，避免 token 数量多的来源在一次全局 softmax 里天然占优势。然后 Source Gate 根据当前视觉内容比较四个来源，决定这个位置更应该相信哪一种信息。得到融合 context 以后，FiLM 产生 gamma 和 beta，分别对视觉通道做缩放和平移；Spatial Gate 再决定当前空间位置和月份接受多少调制，最后通过 residual strength 写回 decoder 前特征。
>
> E9 和 E10 是最重要的受控架构对照：两者都用 3D-Aware 和 M1 到 M4，唯一变化是 CA-HPI 换成 SA-SFiLM。E10 的 mIoU 是 0.60073，比无先验 B0 高 0.128 个百分点，比四源 CA-HPI E9 高 0.678 个百分点。这说明来源分层和空间—通道调制确实缓解了四源 CA-HPI 的退化。
>
> 但 E10 仍然低于只使用 M1 的 E7，也没有形成很大的总体增益。所以我们不能把它讲成“V3 已经全面超过 V2”。更准确的结论是：融合架构会改变多源先验的边际收益，先验的作用仍然具有类别选择性；与此同时，我们已经把从先验表示、来源选择、特征调制到诊断的完整模式搭起来了。

### 过渡

> 到这里，项目的两条成果主线已经清楚了。最后一页总结已有成果，并把可能追加的小尝试留在明确的位置上。

### 建议讲稿

> 最后把项目成果分成两条线来看。第一条是性能和任务线：我们从输入协议和 token 语义开始，逐步比较空间读取和时间融合，最后用 3D-Aware 得到 0.59945 的强视觉基线。第二条是方法线：我们把物候、气候、土壤和地理信息统一成 PriorBatch，先做 Global Add，再做内容感知的 CA-HPI，最后做来源感知的 SA-SFiLM，并用 E7 到 E10 的总体和逐类结果说明它的收益边界。
>
> E10 没有带来很大的总体提升，但这不等于后半阶段没有成果。我们已经回答了一个更具体的问题：多源先验放在同一个 attention 里会发生什么，重新设计来源选择和特征调制以后能改善什么、还剩下什么问题。结果告诉我们先验并不是无条件加分，但接口、融合方式和诊断流程已经形成了一套可复用的模式。
>
> 后面可能还有两个小尝试。一个是检查 M2 气候数据的空间分辨率和 tile 对齐，另一个是尝试加入分类层级。如果结果足够稳定、能够解释当前问题，我们再把它们补进结项材料；如果没有形成可靠证据，就把它们作为明确的后续方向，不改变当前两条成果主线。

### 结尾表述

> 总体来看，本项目一方面建立了从 Galileo 时空表征到作物像素级分割的可靠读取链路，并通过时间融合获得了明确的性能结果；另一方面，围绕异构先验完成了从固定广播到内容感知、再到来源感知调制的连续探索。最终得到的不是一个简单的“加先验后涨分”的结论，而是一套能够描述先验、接入先验、选择先验并诊断效果的模式，同时也明确了它在强视觉基线和多源知识条件下的适用边界。

## 后半部分展示纪律

- V1 的 P0/P1/P2 必须注明是 Single Temporal Readout，不能和 3D-Aware B0 混为一组；
- CA-HPI 与 SA-SFiLM 要使用同一个视觉 token 贯穿讲解，避免听众误以为是两套完全不同的输入；
- E9/E10 是最严格的架构对照，强调两者只改变融合方式；
- E10 不能写成“全面提升”，应写成“相对四源 CA-HPI 有改善，但低于 M1-only”；
- 总体结果和逐类结果都使用表格，不制作柱状图；
- F1/F2 没有正式结果前，只显示为预留位，不放假设性数值；
- 先验模式的通用性指接口、融合和诊断方式可复用，不表述为已经完成跨任务泛化验证。
