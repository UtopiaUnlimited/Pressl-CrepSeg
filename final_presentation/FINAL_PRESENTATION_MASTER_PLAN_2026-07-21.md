## 1. 汇报定位

建议题目：

> **从遥感基础表征读取到异构先验注入**
> 基于 frozen Galileo 的 PASTIS 作物时序分割研究

这次汇报不能只讲最后一次先验实验，也不能把所有实验按日期堆叠。完整项目应被组织为
一条由研究问题推动的递进路线：

```text
Galileo 已经包含可迁移的遥感时空信息吗？
  -> 如何把 Transformer token 可靠地读成像素级作物分割？
  -> 空间 decoder 之外，下游是否过早压缩了时间信息？
  -> 建立强时间基线后，外部知识还能否提供独立信息？
  -> 单一先验有弱正信号时，多源异构先验为什么反而退化？
  -> 如何沉淀为一套可复用、可诊断的先验注入模式？
```

全篇的核心论点是：

> 本项目围绕 frozen 遥感基础模型的下游读取问题，先完成可信的数据与空间解码链路，
> 再验证时间结构是主要性能来源，最后进一步探索异构外部知识的统一表示与内容感知
> 注入。最终成果既包括 3D-Aware 时间融合取得的性能结果，也包括一套从先验表示、
> 模块接入到效果诊断的通用先验注入模式。

## 2. 三层成果定义

| 层次 | 项目成果 | 证据 |
| --- | --- | --- |
| 基础与工程层 | 对齐 Galileo 论文输入协议，正确恢复时空 token，建立 frozen feature 缓存、训练、评估与审计链路 | 输入修正后可稳定完成 19 类分割；统一 folds、缓存和 checkpoint 规则 |
| 任务与性能层 | 证明空间解码和时间读取方式显著影响 Galileo 表征迁移，形成 3D-Aware 强基线 | 同协议 Linear 32.09%，最终层卷积 46.02%，多层同尺度 48.01%，3D-Aware 59.945% |
| 方法与研究层 | 建立统一异构先验接口，并依次探索 Global Add、CA-HPI 与 SA-SFiLM | `PriorBatch`、M1–M4 adapters、V1 反事实、V2 多源结果、V3 来源感知调制与诊断 |

结项时不要把“精度结果”和“方法模式”混成同一件事：时间融合是项目最明确的性能成果；
先验体系是项目后半阶段的方法成果。V3 若有提升，会增强方法有效性证据；若提升有限，
统一接口、两种融合架构、反事实设计和诊断流程仍然构成完整研究产出。

### 第 2 页：研究背景——为什么需要时序作物制图

页面标题：**大范围农业监测需要持续、精细的作物类型信息**

| 农业信息需求 | 传统获取限制 | 卫星时间序列提供的机会 |
| --- | --- | --- |
| 作物种植结构与空间分布 | 田间调查成本高、更新频率有限 | 大范围、重复观测 |
| 地块尺度类别与边界 | 汇总统计难以形成连续空间结果 | 像素级制图与区域统计 |
| 生长季动态变化 | 单次调查或单景影像容易遗漏过程 | 多时相光谱记录生长轨迹 |

口头主旨：作物制图不是普通单图分类，而是卫星影像时间序列上的像素级识别问题。

### 第 3 页：研究意义——为什么要研究冻结基础模型的下游读取

页面标题：**把通用遥感表征真正转化为农业像素级结果**

| 意义层次 | 本项目希望解决什么 |
| --- | --- |
| 应用意义 | 让大范围、低成本的作物类型制图更充分利用多时相遥感信息 |
| 科学意义 | 研究冻结遥感基础模型的多层、多时间表征如何迁移到 dense prediction |
| 方法意义 | 将物候、气候、土壤和地理等异构知识整理为可插拔、可诊断的先验接口 |

方法演进带：

```text
任务专用时序网络 -> 遥感自监督预训练 -> Galileo 等基础模型
                                             -> 仍需解决下游 token 读取与先验注入
```

### 第 4 页：研究内容——围绕空间、时间和先验逐层推进

页面标题：**研究问题来自基础模型与像素级任务之间的三个缺口**

```text
Galileo：输出多层、多时间 Transformer token
                         ↓
PASTIS：要求与地块边界对齐的 19 类像素图
                         ↓
缺口：token 如何恢复空间？月份何时融合？外部知识如何补充？
```

| 层次 | 核心问题 | 项目中的验证路线 |
| --- | --- | --- |
| RQ1 空间读取 | frozen Galileo token 能否被可靠恢复为像素级结果？ | Linear → 最终层卷积 → 多层融合 |
| RQ2 时间读取 | 固定平均是否过早丢失月份身份？ | Temporal Readout → 3D-Aware |
| RQ3 知识注入 | 强视觉模型之后，异构先验能否提供独立信息？ | Global Add → CA-HPI → SA-SFiLM |

### 第 5 页：研究流程与实验基础

页面标题：**先建立可信基线，再逐步增加时间结构和外部知识**

```text
研究背景与任务定义
  -> 对齐 PASTIS—Galileo 输入协议
  -> 空间读取：Linear / 卷积 / 多层 decoder
  -> 时间读取：Temporal Readout / 3D-Aware
  -> 先验注入：Global Add / CA-HPI / SA-SFiLM
  -> 总体与逐类结果、诊断和适用边界
```

```text
PASTIS [T,10,128,128]
  -> 2018-10 至 2019-09，聚合为 12 个月
  -> 每张切为 4 个 64×64 tile
  -> frozen Galileo，多层、多月份时空特征
  -> 19 类 64×64 segmentation
```

| 项目 | 固定设置 |
| --- | --- |
| 数据划分 | train folds 1/2/3，val fold4，test fold5 |
| Galileo | ViT-Base，完全冻结 |
| 模型选择 | fold4 最高 `val_mIoU` checkpoint |
| 主指标 | mIoU，并同时报告 accuracy、macro F1 和逐类结果 |

本页同时保留三项决定可信度的修正：对齐作物年与月度输入；按空间、时间和通道组恢复
Galileo token；建立 `spatial_v1` 与 `temporal_v2` 共享缓存。项目早期的低结果首先触发
协议审计，而不是直接增加网络复杂度。

### 第 6 页：空间解码路线与第一阶段结果

页面标题：**空间细化和多层表征显著优于线性读取**

| 方法 | 可训练参数 | Test mIoU | 回答的问题 |
| --- | ---: | ---: | --- |
| 同协议 Linear Head | 0.234M | 32.09% | 最终层是否线性可分 |
| 最终层卷积 | 4.925M | 46.02% | 空间细化是否必要 |
| 多层同尺度融合 | 9.058M | 48.01% | 中间层是否提供补充 |

页面结论：最终层卷积相对 Linear 提升 `13.92 pp`，多层融合再提升 `1.99 pp`。这证明
Galileo frozen feature 包含可迁移信息，但下游读取结构是决定性能的关键变量。

同时保留边界：参数量不同、只有 seed42，因此这是完整方案比较，不是单位参数效率证明。

### 第 7 页：发现真正的时间瓶颈

页面标题：**旧接口保留了时间上下文，却过早丢失月份身份**

```text
12 个月共同进入 Galileo
  -> encoder 内已有跨时间交互
  -> F [B,L,T,D,H,W]
  -> 固定 mean(T)
  -> G [B,L,D,H,W]
  -> 2D decoder
```

需要讲清楚：旧方案并非完全没有时间信息；问题出在 encoder 与 decoder 的边界——固定
平均使 decoder 无法针对不同层、位置和月份学习差异化权重。

### 第 8 页：两种时间读取方式

页面标题：**从固定时间平均推进到可学习时间融合**

| 路线 | 时间维保留到哪里 | 特点 |
| --- | --- | --- |
| Temporal Readout | decoder 前聚合 | 每层、每个空间位置学习月份权重，可接任意 2D decoder |
| 3D-Aware DPT | decoder 高分辨率融合后再聚合 | 时空与跨层联合处理，时间维保留更久 |

本页使用上下两条平行数据流。不要把 Temporal Readout 与 3D-Aware 画成完全无关的模型；
它们共同回答“何时压缩时间维”这一研究问题。

### 第 9 页：时间建模取得项目最明确的性能成果

页面标题：**保留时间结构后，mIoU 提升到约 60%**

建议使用两张表，不使用柱状图。

表一：四种 Temporal Readout 的同协议 fold5 结果。

| 2D decoder + Temporal Readout | Test mIoU | Test F1 |
| --- | ---: | ---: |
| Single | 0.54938 | 0.68833 |
| Multi | **0.56154** | **0.69712** |
| Galileo-Adapted DPT | 0.54637 | 0.68433 |
| UPerNet | 0.54442 | 0.68279 |

表二：3D-Aware 的最终性能定位。

| 方法 | Test loss | Test mIoU | Test accuracy | Test F1 |
| --- | ---: | ---: | ---: | ---: |
| 3D-Aware，无先验 | 1.03279 | **0.59945** | 0.81781 | 0.73133 |

口头结论：Temporal Readout 证明“可学习月份读取”普遍可行；3D-Aware 进一步表明，在
空间融合过程中持续保留时间维具有更强表现。它是后续先验研究的固定强基线。

### 第 10 页：总体提升之外，哪些类别仍然困难

页面标题：**时间融合改善多数类别，但类别异质性仍然存在**

本页使用精简逐类表，只保留代表性类别，不放完整 19 行。

| 类别组 | 代表类别 | 3D-Aware B0 IoU | 观察 |
| --- | --- | ---: | --- |
| 高表现季节作物 | Corn / Rapeseed / Beet | 0.854 / 0.884 / 0.858 | 时序视觉特征已经很强 |
| 仍较困难类别 | Sorghum / Leguminous fodder / Winter triticale | 0.312 / 0.365 / 0.368 | 少样本、相似类别或标签异质性仍未解决 |
| 多年生或聚合类别 | Orchard / Mixed cereal / Fruits-vegetables-flowers | 0.408 / 0.398 / 0.363 | 单一标准物候曲线不一定适用 |

页面结论：强时间基线压缩了显式物候先验的可提升空间，也提示先验是否有效不能只看总体
mIoU，而要看类别选择性和知识适用边界。

### 第 11 页：V1 固定物候广播及其反事实结果

页面标题：**正确物候出现弱信号，但全局广播难以充分利用知识**

先简述 V1：`19×12` 物候表经 MLP 投影后，在对应月份广播到所有空间位置。

| 实验 | Test mIoU | 相对无先验 |
| --- | ---: | ---: |
| P0 Single，无先验 | 0.54938 | - |
| P1 正确物候 | **0.55316** | +0.378 pp |
| P2 类别置乱物候 | 0.54784 | -0.154 pp |

需要讲出科研判断：P1 同时高于 P0 和 P2，说明知识内容可能存在弱信号；但提升很小、类别
收益不稳定，而且同一残差广播给所有像素过于粗糙，因此研究问题从“是否加入一张物候
表”转向“如何让视觉内容选择知识”。

### 第 12 页：从单一物候表抽象为统一异构先验体系

页面标题：**先验数据形态不同，但可以共享同一接口**

| 来源 | 数据形式 | token 数 | 科学边界 |
| --- | --- | ---: | --- |
| M1 物候 | 类别×月份的强度与置信度 | 228 | 聚合类、多年生类允许低 confidence 或 unknown |
| M2 ERA5-Land | patch×月份的温度、降水、辐射、表层水分 | 12 | 区域天气背景，不是地块实测 |
| M3 SoilGrids | 三个深度的土壤属性与不确定性 | 3 | 250 m 模型预测，不是地块采样 |
| M4 Geography | train-fold 标准化经纬度 | 1 | 可能利用区域共现，需要警惕空间捷径 |

统一接口：`PriorBatch = tokens + mask + confidence + type/source id`。

口头结论：通用性首先来自数据接口和插入位置，而不是宣称已经在其他任务上证明泛化。
所有统计只使用外部资料或 train folds，fold4 选模型，fold5 不反向修改先验。

### 第 13 页：V2 CA-HPI——内容感知的异构先验注入

页面标题：**让每个视觉时空位置主动查询完整先验库**

建议按一个视觉 token 的数据流讲六步：

```text
视觉时空 token -> Query
先验 token 集合 -> Key / Value
mask + confidence + source balance -> attention
加权得到候选残差 -> content gate
逐层 strength 控制注入幅度
decoder 前写回原特征金字塔
```

核心区别：V1 是同月全局广播；CA-HPI 是每个位置、月份和层根据视觉内容选择知识，并
允许低置信度、缺失项和无效先验被屏蔽。

### 第 14 页：V2 多源结果揭示“来源越多不一定越好”

页面标题：**单一物候有弱正信号，多源组合出现系统性干扰**

| 实验 | 先验 | Test mIoU | 相对 B0 |
| --- | --- | ---: | ---: |
| B0 | 无 | 0.59945 | - |
| E7 | M1 | **0.60443** | **+0.498 pp** |
| E8 | M1+M2+M3 | 0.60192 | +0.247 pp |
| E9 | M1+M2+M3+M4 | 0.59395 | -0.550 pp |

逐类证据只保留最能解释机制的类别：

- Grapevine：M1 `+4.82 pp`，说明部分困难类别能够利用物候知识；
- Orchard：M1234 `+4.01 pp`，地理/环境 context 可能有帮助；
- Sorghum：M1 `+3.08 pp`，但 M1234 `-9.25 pp`，对来源组合高度敏感；
- Leguminous fodder 与 Soybeans 在多源组合中明显下降。

研究判断：问题不再是“先验有没有接入”，而是不同来源 token 数量、适用对象和信息尺度
不同，单次全局 softmax 使它们发生竞争。总体 mIoU 的小变化实际上由明显的类别正负变化
共同形成。

### 第 15 页：V3 SA-SFiLM——从全局竞争到分层选择

页面标题：**先在来源内选知识，再决定来源、通道和空间位置**

```text
每个来源内部 attention -> context_s
视觉内容 + context_s -> source gate
多来源 context 加权融合
context -> FiLM gamma / beta
spatial gate -> 控制位置和月份
decoder 前残差写回
```

与 CA-HPI 的严格对照应固定：3D-Aware、M1/M2/M3/M4、训练协议和 decoder 前插入点；
只改变融合架构。

本页最终需要补入：

| 实验 | Test loss | Test mIoU | 相对 B0 | 相对 E9 | Test F1 | 主要诊断 |
| --- | ---: | ---: | ---: | ---: | ---: | --- |
| E10 SA-SFiLM M1234 | 1.22321 | 0.60073 | **+0.128 pp** | **+0.678 pp** | 0.73051 | source weights、FiLM scale/shift、applied residual 待补 |

讲稿结论：

> E10 相比无先验 B0 的 mIoU 提升 0.128 个百分点，说明四源 SA-SFiLM 至少没有破坏强
> 视觉基线；相较 CA-HPI 的四源 E9 提升 0.678 个百分点，支持来源分层和空间—通道调制
> 比简单多源竞争更合理。但它仍低于 CA-HPI 的 M1-only E7（0.60443），因此不能把
> SA-SFiLM 描述成已经全面超过 CA-HPI。更准确的结论是：我们完成了一个可运行、可诊断的
> 通用先验注入模式，并获得了“融合架构会影响多源先验边际收益”的实验证据。

### 第 16 页：结项成果、边界与最终认识

页面标题：**项目最终形成了两条相互衔接的成果主线**

| 主线 | 最终成果 | 科研认识 |
| --- | --- | --- |
| 表征读取与时间融合 | 可信输入链路、多类 decoder、Temporal Readout、3D-Aware 强基线 | frozen foundation feature 的价值取决于下游如何恢复空间并保留时间结构 |
| 异构先验注入 | M1–M4 数据规范、PriorBatch、CA-HPI、SA-SFiLM、反事实与诊断体系 | 先验不是无条件增益；知识内容、来源竞争和强视觉基线共同决定收益边界 |

结项前可选的小尝试只预留两个位置，不预填结果：

| 预留位 | 可能内容 | 当前状态 |
| --- | --- | --- |
| F1 | 完善 M2 气候先验的空间分辨率与 tile 对齐 | 待决定，无结果 |
| F2 | 增加分类层级 head 或 coarse-to-fine 分类路径 | 待决定，无结果 |

只有在形成稳定且可解释的结果后，F1/F2 才进入主汇报；否则放在后续工作或备份页。

建议结尾表述：

> 本项目没有把所有提升简单归因于某一个模型，而是沿着实验暴露的问题逐步拆分变量。
> 我们首先建立了能够可靠读取 Galileo 时空表征的分割链路，并通过 3D-Aware 时间融合
> 获得约 60% mIoU；随后进一步把物候、气候、土壤和地理信息统一为可学习先验，完成
> 从全局广播到内容感知、再到来源感知空间—通道调制的连续探索。最终结果既给出了
> 先验能够帮助哪些类别的证据，也揭示了强时序模型下多源知识的边际收益和适用边界。

结尾不需要强调“停止修改”或“进入收尾”。重点是：性能结果回答了任务问题，统一先验
模式回答了方法问题，两者共同构成项目成果。

## 6. 汇报语言与展示纪律

- 不说“达到了 SOTA”或把不同协议数字直接排名；
- 不把 3D-Aware 的全部提升只归因于时间融合，因为结构和容量也变化；
- 不把旧 Single + Global Add 的 P1/P2 说成 3D-Aware 结果；
- 不把 M2/M3/M4 称为地块实测数据；
- 不把接口通用扩大成“已经证明跨任务泛化”；
- 不因 E10 的总体提升有限，就把先验模式描述成失败；应同时报告架构对照和适用边界；
- 所有总体结果使用表格，不制作柱状图；分类别结果只在主页面保留能够解释机制的代表类；
- 每页只保留一个研究结论，详细配置、命令和全量表格放备份页。
