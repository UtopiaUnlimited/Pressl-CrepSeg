## 3. 目录结构

```
~/                     # SSD，空间小 → 只放 .bashrc 等配置
~/local/               # 外挂机械硬盘 → 所有工作放这里
  conda_envs/          # conda 环境目录
    presl/             # 本项目环境
  pressl-cropseg-mirror/  # 项目代码
    data/cache/galileo-base-patch8/  # 特征缓存
    logs/              # TensorBoard 日志

    runlogs/           # 训练 stdout 日志
```

## 4. 环境激活

```bash
conda run -n presl python -B scripts/xxx.py ...
```

环境名 `presl`，Python 3.10，PyTorch 已装好。

## 5. 常用命令速查

### 看 GPU

```bash
nvidia-smi
```

### 看谁在跑什么

```bash
ps aux | grep train_cached | grep -v grep
```

### 杀训练进程

```bash
# 先确认再杀（每个训练有两个进程：conda wrapper + python）
kill <conda_pid> <python_pid>
```

### 启动 TensorBoard

```bash
cd ~/local/pressl-cropseg-mirror

# 前台（调试用）
conda run -n presl tensorboard --logdir logs --bind_all --port 6006

# 后台
nohup conda run -n presl tensorboard --logdir logs --bind_all --port 6006 > /dev/null 2>&1 &
```

浏览器访问 `http://10.130.11.69:6006`（端口被占就换 6007、6008）。

### 确认端口占用

```bash
ss -tlnp | grep 6006
```

## 6. 训练命令

### 冒烟测试（1 epoch 2 batch，验证 pipeline 不报错）

```bash
cd ~/local/pressl-cropseg-mirror

conda run -n presl python -u -B scripts/train_cached.py \
  --config configs/galileo_single_layer_dpt_temporal_readout.yaml \
  --train-cache-dir data/cache/galileo-base-patch8/monthly12_tile64_patch4_hl3-6-9-12_temporal-v2_tfp16_train \
  --val-cache-dir data/cache/galileo-base-patch8/monthly12_tile64_patch4_hl3-6-9-12_temporal-v2_tfp16_val \
  --cache-format temporal_v2 --temporal-dtype float16 \
  --max-train-batches 2 --max-val-batches 2 --epochs 1 \
  --device cuda:0
```

## 7. 并行训练注意事项

- **双 GPU 可并行**：`--device cuda:0` 和 `--device cuda:1` 各跑一个实验
- **不要同卡跑多个**：OOM 风险，每个单层 DPT 约 6-7GB，多进程 I/O 争抢机械盘也慢
- **启动后验证**：`nvidia-smi | grep python` 确认显存在涨（不是卡在初始化）
- **TensorBoard events 文件 >500 字节** 才说明真正在跑

## 8. 常见故障排查

| 现象 | 原因 | 解决 |
|------|------|------|
| TensorBoard 连不上 | 端口被占 | 换 `--port 6007`，用 `ss -tlnp` 查占用 |
| events 文件 88 字节 | 训练死在第一个 epoch 前 | OOM 或 DataLoader 崩，换空闲 GPU 重来 |
| 日志只有 "nohup: ignoring input" | bash -c 吞 stdout | 正常，看 TensorBoard 就行 |
| 显存不动（<1GB 持续很久） | 数据加载慢 | 机械盘 I/O 瓶颈，等或减少并发进程 |
| 多个旧进程堆积 | 前次 nohup 没杀干净 | `ps aux \| grep train_cached`，kill 对应 PID |
| conda 找不到包 | 无外网 | 配浙大源或用 `pip install -i https://mirrors.zju.edu.cn/pypi/simple` |

## 9. 协作公约

1. **动前先看 GPU 占用**：`nvidia-smi` 确认不抢别人正在跑的卡
2. **跑完或停实验及时 kill 进程**：别占着显存不做事
3. **TensorBoard 统一用 `--logdir logs`**：所有人共享同一个 logs 目录，选择不同端口
4. **不要往 `~/` 放大文件**：SSD 空间有限
5. **模型和数据放 `~/local/`**：外挂硬盘，空间大
6. **命名要可追溯**：nohup 日志 > `logs/`，checkpoint 自动按 config+phenology 分目录
