# Seasonal Contrast：遥感预训练优势的论文翻译与解读

> 论文：Seasonal Contrast: Unsupervised Pre-Training from Uncurated Remote Sensing Data  
> 作者：Oscar Mañas 等  
> arXiv：<https://arxiv.org/abs/2103.16607>  
> 官方代码：<https://github.com/ElementAI/seasonal-contrast>

这篇论文研究的核心问题是：遥感数据很多，但标注很少，而且遥感影像天然带有“同一地点、不同时间”的观测。能不能利用这些未标注的多时相影像，先学出一个适合遥感的视觉表示，再迁移到分类、分割和变化检测等任务？

论文给出的答案是 Seasonal Contrast（SeCo）：利用同一地理位置不同季节的影像作为天然的正样本，同时把“应该忽略的变化”和“应该保留的变化”分配到不同的表示子空间中。

---

## 1. 论文想解决什么问题

### 1.1 标注数据是瓶颈

地球观测卫星每天产生大量影像，但像素级标注通常需要专家知识，成本很高。传统监督学习只能使用其中很小的一部分。

自监督预训练的思路是先利用没有人工标签的数据学习表示，再把这个表示迁移到有标签的下游任务。对于遥感来说，直接使用 ImageNet 预训练并不一定合适，因为自然图像和卫星图像之间存在明显的领域差异：

- 传感器和光谱通道不同；
- 俯视视角与自然图像不同；
- 地物尺度、纹理和颜色分布不同；
- 遥感数据具有空间位置和时间序列信息。

因此，这篇论文的基本假设是：**遥感预训练数据和预训练目标都应该尽量利用遥感自身的结构。**

### 1.2 时间变化既可能是噪声，也可能是信息

同一个地点在不同季节的影像，通常仍然属于相同的地表区域或土地覆盖类型，但外观会发生变化。例如农田的颜色、植被覆盖度和纹理会随季节变化。

于是，时间变化有两种性质：

- 对土地覆盖分类或作物类型分割来说，季节变化有时是需要适应的变化，模型不能因为季节不同就认为地物完全不同；
- 对变化检测来说，时间变化本身可能就是任务目标，模型不能把所有时间差异都消除。

SeCo 的关键贡献，就是没有强行让所有特征都对时间不变，而是学习同时包含时间不变信息和时间变化信息的表示。

## 2. 预训练数据：不依赖人工精选的数据

作者从 Sentinel-2 影像中自动构建了一个未标注数据集：

- 大约 200,000 个地理位置；
- 每个位置采集 5 个不同日期的影像；
- 时间间隔大约为 3 个月，以覆盖季节变化；
- 云量超过 10% 的影像被过滤；
- 最终约有 100 万个多光谱影像块；
- 每个影像块覆盖约 2.65 × 2.65 km。

采样位置不是在地球表面完全均匀随机采样，而是围绕人口较多的城市采样。作者的理由是：城市附近不仅包含建筑，也常常能覆盖农田、工业设施、森林和自然环境，能够在有限数据量下获得更丰富的地表类型。

这里有一个重要的研究启发：**预训练数据的价值不只取决于数量，也取决于它是否包含与下游任务相匹配的变化因素。**

## 3. Seasonal Contrast 的核心方法

### 3.1 正样本不再只是人工增强

普通对比学习通常把同一张图片经过两种随机增强后得到的结果作为正样本。例如裁剪、颜色扰动、翻转后的两张图被认为语义相同。

SeCo 除了使用人工增强，还使用同一地理位置的不同时间影像作为“季节正样本”：

```text
同一位置、时间 t0 的影像：query
同一位置、时间 t1 的影像：seasonal key
同一位置、时间 t2 的影像：seasonal key
```

它利用了遥感数据中天然存在的“同位置重复观测”。这种正样本比单纯的颜色抖动更有意义，因为它展现了真实的季节变化。

### 3.2 三个表示子空间

设编码器输出一个共享表示 `V`，然后通过三个投影头得到三个子空间 `Z0`、`Z1` 和 `Z2`。三个子空间承担不同的不变性约束：

| 子空间 | 对什么不变 | 对什么保持变化 | 直观含义 |
|---|---|---|---|
| `Z0` | 人工增强、季节变化 | 不强调上述变化 | 最稳定的语义表示 |
| `Z1` | 季节变化 | 人工增强 | 适合关注地物身份而不关注季节外观 |
| `Z2` | 人工增强 | 季节变化 | 保留时间变化信息 |

这里的重点不是三个空间本身，而是共享编码器必须同时保留多种因素。最终迁移到下游任务时，使用的是共享表示 `V`，而不是提前假设某一个下游任务只需要某一种不变性。

### 3.3 视图和正负样本关系

对于一个 query，作者构造三个 key：

```text
xq  = t0 时刻的原始影像
xk0 = t1 时刻影像 + 人工增强
xk1 = t2 时刻影像
xk2 = t0 时刻影像 + 人工增强
```

三个子空间中的正负关系不同：

- `Z0`：所有 key 都可以作为 query 的正样本，因为它希望同时忽略季节变化和人工增强；
- `Z1`：只有季节变化后的 key 是正样本，人工增强相关的视图作为负样本；
- `Z2`：只有人工增强后的 key 是正样本，季节变化后的视图作为负样本。

因此，SeCo 并不是简单地把“同位置不同时间”全部拉近，而是明确控制：哪些空间应该具有时间不变性，哪些空间应该保留时间变化。

### 3.4 对比学习损失

各个子空间分别使用 InfoNCE 类对比损失，最终目标是三个损失之和：

```text
L_SeCo = L_Z0 + L_Z1 + L_Z2
```

对比学习的直觉是：把正样本的表示拉近，把负样本的表示推远。论文中使用 MoCo-v2 的动量编码器结构，以降低对比学习对大 batch size 的依赖。

## 4. 实验结果说明了什么

作者在三个下游任务上评估 SeCo：BigEarthNet 土地覆盖分类、EuroSAT 土地覆盖分类，以及 OSCD 变化检测。

### 4.1 BigEarthNet 分类

在 BigEarthNet 上，SeCo 的平均精度优于随机初始化、ImageNet 监督预训练、MoCo-v2，以及加入时间正样本的 MoCo-v2+TP。以 ResNet-18、100 万预训练影像为例：

- 10% 标注数据线性探测：SeCo 约 76.05%，ImageNet 约 65.69%；
- 100% 标注数据线性探测：SeCo 约 77.00%，ImageNet 约 66.40%；
- 100% 标注数据微调：SeCo 约 87.27%，ImageNet 约 85.90%。

这说明遥感领域的自监督预训练不仅在小标注数据场景下有帮助，即使有更多标注，它仍然能提供更好的初始化。

### 4.2 EuroSAT 分类

SeCo 在 EuroSAT 上也优于 ImageNet 和普通 MoCo-v2，说明它学到的表示并不只适用于预训练数据本身，而可以迁移到另一套遥感数据集。

### 4.3 OSCD 变化检测

变化检测是论文中最能说明问题的实验。SeCo 的 F1 分数高于所有比较方法，约为 46.94%，而 MoCo-v2+TP 约为 40.12%。

这里值得注意的是：如果把同一位置不同时间的影像简单地当成必须拉近的正样本，模型可能会过度消除时间差异，这对变化检测不利。SeCo 通过多个子空间同时保留时间不变和时间变化因素，因此在变化检测上表现更好。

### 4.4 论文真正证明的不是“预训练一定更强”

更准确的结论是：

> 与下游任务相关的领域数据，加上与该领域结构匹配的自监督目标，通常比直接套用 ImageNet 预训练更有价值。

这里包含三个条件：

1. 预训练数据确实覆盖了下游任务需要的变化因素；
2. 正负样本的构造没有错误地抹除任务所需的信息；
3. 下游模型能够读取预训练表示中的有效信息。

## 5. 对我们 Galileo + PASTIS 项目的启发

### 5.1 Galileo 预训练的价值来自“领域和结构匹配”

我们的任务输入是多时间、多光谱的 Sentinel-2 数据，目标是像素级作物分割。Galileo 在大规模多模态遥感数据上进行预训练，并且专门设计了全局和局部的自监督目标，因此它比普通自然图像预训练更有理论上的适配性。

我们的 baseline 其实在检验一个很清晰的问题：

> Galileo 的冻结表示是否已经包含足够的时空和光谱信息，使一个相对简单的 DPT 解码器能够恢复作物类别的空间分布？

这比一开始就修改很多结构更有科研价值，因为它先测量预训练表示的可迁移性。

### 5.2 作物分割需要“稳定信息”和“时间信息”同时存在

PASTIS 的作物类别通常与地块的长期结构、光谱模式和空间上下文有关，但作物在不同时间的外观也会发生真实变化。因此，我们不能简单地认为：时间变化全是噪声，或者时间变化全是目标。

这与 SeCo 的观点一致：好的表示应该同时保留两类因素。

在我们的项目里，它们可能对应：

- 稳定因素：地块形状、空间邻域、长期作物类型、光谱组合；
- 时间因素：生长阶段、植被覆盖度、收获前后的外观变化。

这也解释了为什么 Galileo 的多时相建模有意义：模型不只是看一张影像，而是利用整个时间序列来判断像素属于哪一类作物。

### 5.3 为什么 baseline 先用最终层是合理的

SeCo 使用共享表示来进行迁移，而不是在预训练阶段直接针对某一个下游任务设计分类器。我们当前的 baseline 使用 Galileo 最终输出的空间特征，再接单层 DPT 解码器，实际上是在做表示迁移的第一步验证。

如果这个 baseline 已经取得稳定结果，就说明：

- Galileo 最终表示具备一定的任务可读性；
- token 到空间特征图的重组是可用的；
- DPT 解码器能够将语义表示恢复为空间分割结果。

此后再引入 Galileo 第 3、6、9、12 层的多层 hidden states，才可以研究一个更具体的问题：

> 作物边界和局部细节是否主要存在于较浅层，而最终层的语义表示是否需要与中间层融合？

这会是有明确假设的扩展实验，而不只是“堆更多层”。

### 5.4 需要警惕时间信息被错误平均

SeCo 论文最重要的提醒之一是：时间关系不能被粗暴处理。

我们当前对 Galileo 的结构化 space-time token 在时间维度和通道组维度上做聚合，得到空间网格特征，这是为了让 DPT 解码器能够工作。但这个操作本质上会压缩时间信息。

因此 baseline 的科学解释应该是：

> 测试“经过 Galileo 时空融合后的空间表示”是否足以完成分割。

它不能被解释为已经充分利用了时间序列的全部信息。后续可以考虑的方向包括：

- 对不同时间步使用可学习加权，而不是简单平均；
- 使用时间注意力汇聚后再输入 DPT；
- 保留多个时间尺度的特征；
- 比较只使用单时相、时间平均、时间注意力和完整时序建模。

这些实验必须逐步进行，否则很难知道性能变化来自时间建模、解码器容量还是训练策略。

## 6. 这篇论文的局限性

### 6.1 SeCo 并不等于 Galileo

SeCo 主要基于 CNN 和全局影像级对比表示，输入形式也比 Galileo 简单。它不能直接说明 Galileo 的 token 聚合、DPT 解码或多层 hidden states 应该如何实现。

它对我们的价值主要是提供研究动机：为什么遥感预训练应该利用时间结构，以及为什么不能随意定义时间不变性。

### 6.2 “同位置不同时间”不一定总是正样本

对于土地覆盖分类，同一地点的季节影像通常可以作为相近语义的样本；但如果研究变化检测、灾害过程或作物生长阶段，时间差异本身可能是目标。

所以正样本定义必须由下游任务决定。不能把“同一个地理位置”机械地等价成“所有语义都相同”。

### 6.3 数据采样启发式可能带来分布偏差

围绕城市采样有助于提高地表类型多样性，但也可能使数据分布偏向人口密集区域。预训练数据越大，并不意味着它一定代表所有地理区域。

---

## 7. 用一句话总结

Seasonal Contrast 证明了：**遥感预训练的优势不只是“使用了更多数据”，而是利用了遥感独有的时间重复观测，并通过不同表示子空间同时保留时间不变和时间变化的信息。**

对我们的研究而言，它支持 Galileo 作为遥感时空预训练编码器的选择，也提醒我们：baseline 应先验证最终表示能否迁移到作物分割；之后的多层 DPT 和时间聚合实验，则应围绕“局部边界、稳定作物语义、季节变化信息分别在哪里”提出可检验的假设。

## 8. 建议的阅读顺序

如果英文阅读比较吃力，可以按下面顺序读原文：

1. 先读 Abstract，理解论文结论；
2. 读 Introduction 的最后两页，弄清楚作者为什么不直接使用单一时间不变表示；
3. 读 Figure 2 和 Section 3.2，重点看三个子空间的正负样本关系；
4. 读 Table 1 和 Table 4，比较分类与变化检测中不同预训练方法的差异；
5. 最后读 Conclusion，把论文结论和我们的 Galileo baseline 对照起来。
